## 四、业务架构优化

### 4.1 五司重映射（28 → 五司）
基于各 skill 的实际 `alliance.department` 与职能，建议归位：

| 司 | 现有真实 skill（示例） |
|----|----------------------|
| **情报司**（洞察） | 行业洞察、客户洞察、商机洞察、客户档案、深度研究、会议纪要、访前准备 |

| **执行司**（赢单） | 信任拜访、解决方案（缺）、销售助理、销售体系构建 |
| **经营司**（客户经营） | 客户经营、客户经营计划、生态伙伴、伙伴诊断、组织设计 |
| **监察司**（管理） | 销售管理、销售教练（缺）、销售日常驾驶舱、浙江渠道周报 |
| **中枢** | alliance-commander、量子蜂群（元） |
| **思维层** | 深度思考、苏格拉底、认知专家 |

### 4.2 三 Pipeline 重梳
- **MTL（市场→线索）**：行业洞察 → 客户洞察 → 线索营销(缺) → 线索分配
- **LTC（线索→现金）**：漏斗构建 → 策略赢单 → 信任拜访 → 解决方案(缺) → 客户经营
- **Ops（日常运营）**：销售日常驾驶舱 / 浙江渠道周报 / 销售管理（监控闭环）

### 4.3 语义层落地
5 个 `semantic` 目录（business / auto-parts / manufacturing / medical / tenant）是"三域语义体系"的底座。缺失则客户洞察专家的精准查询退化。建议**路线 A 补建**或**路线 C 降级**。
