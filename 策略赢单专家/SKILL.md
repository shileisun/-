### 【前置】夏凯价值营销8字诀（赢单策略灵魂）

> **来源**：夏凯价值营销直播逐字稿（2026-06-17）
> **定位**：策略赢单的底层灵魂——在制定赢单策略时，必须先用8字诀锚定方向

#### 目标 → 障碍 → 能力 → 价值

```
① 目标（锚定靶心）
   └─ 这个客户（具体岗位的人）想实现什么目标？
   └─ 不是"公司目标"，是"这个人的目标"（厂长的目标 ≠ IT总监的目标）

② 障碍（诊断问题）
   └─ 什么在阻碍他实现这个目标？
   └─ 技术障碍？流程障碍？资源障碍？组织障碍？
   └─ 关键：问题一定是某个岗位、某个人的问题

③ 能力（赋能客户）
   └─ 我们的产品/方案能给他什么能力，让他跨过这个障碍？
   └─ 不是"我们有什么功能"，而是"我们赋予他什么能力"
   └─ 赋能比喻：你卖棍子（产品），他拿棍子打狼（创造价值）

④ 价值（量化结果）
   └─ 这个能力带来什么可衡量的价值改善？
   └─ 5类价值：潜在价值/锚定价值/成本节约/效率提升/收入增长
   └─ 优先选择：高可控（我方能证明/客户能感知）的价值类型
```

#### 8字诀在赢单策略中的应用

| 策略场景 | 8字诀应用 | 输出 |
|:--------|:---------|:-----|
| **EB（经济购买者）拜访** | 目标=年度业务目标；障碍=管理失控；能力=数据驾驶舱；价值=决策效率↑ | 高层拜访PPT |
| **T-CB（技术评估者）** | 目标=系统稳定/安全；障碍=集成复杂度；能力=开放API；价值=上线周期↓ | 技术方案 |
| **UB（用户）** | 目标=少加班/少跑腿；障碍=信息不透明；能力=移动端审批；价值=效率↑/满意度↑ | 场景演示 |

> **核心原则**：解决方案 = 双方对**确定的问题**一致认可的答案，且能带来可衡量的价值改善。
> 先确认问题，再谈解决方案。问题没共识，方案等于废纸。

### 〇、底层信念体系

**信任公式**：
```
信任 = 知识距离（越大越好） / 心理距离（越小越好）
```
- **知识距离**：专业知识、行业洞见、咨询能力、成功案例
- **心理距离**：真诚度、共情力、为客户着想、关注客户感受

**信任四柱**：形象 → 能力 → 共通点 → 诚恳

**影响信任的五要素**：
1. 形象与风格（外表、着装、亲和力、肢体语言）
2. 经历与经验（取得的成果、成功案例）
3. 专业知识与咨询能力（行业知识、场景知识、知识分享能力）
4. 共通点（共同兴趣、经历、处境、价值观）
5. 诚恳与坦率（不操控、不回避、不隐瞒、不欺骗）

**核心洞察**：
> "客户是感性决策，寻找理性依据加以证明。"
> "放下自我，以同理心进入心境、以利他心构建双赢、以敬畏心尊重人性。"
> "销售就是在和客户互相冲动的过程中，看谁能抑制住自己去关注对方，谁就胜利了。"

### 一、业务结果四阶段理论（v12.1新增）

> **来源**：夏凯《赢单罗盘》+ 逐字稿深度提炼
> **核心价值**：帮助销售人员从"需求混沌"到"共识共赢"，系统化推进客户决策

---

#### 1.1 理论背景

在B2B复杂销售中，客户对"业务结果"的认知是一个**渐进式清晰的过程**。销售人员的核心任务，就是推动客户从"不清"走向"共识"。

**关键洞察**：
- 客户一开始往往**不知道自己不知道**（不清）
- 销售人员容易**假设客户已经明确需求**（盲目推销）
- 只有达到"共识"阶段，客户才会**真正投入资源**（签约）

---

#### 1.2 四阶段定义

| 阶段 | 定义 | 客户状态 | 销售任务 | 示例 |
|:----:|:------|:----------|:--------|:------|
| **不清** | 需求混沌状态（客户自己没想清楚/销售没获取到信息） | "感觉生产线不太顺畅，但具体什么问题多大影响，我们没有梳理过" | **探索唤醒**：通过提问帮助客户意识到问题 | 提问："您当前生产线的OEE是多少？行业平均是多少？" |
| **常理** | 基于行业共性角色常理和推理推测 | "一般制造业总监都会关注生产效率" | **验证假设**：用行业基准验证客户情况 | 分享："根据我们的行业调研，类似企业通常关注XX指标" |
| **明确** | 客户亲口表达具体需求（隐性→显性） | "我们需要提升设备OEE 15%" | **量化澄清**：帮助客户将模糊需求转化为可量化指标 | 确认："您说的'提升效率'具体是指OEE从85%提升到多少？" |
| **共识** | 销售方与客户就业务结果形成共同理解和认可 | 不仅表达需求，还就合理性/路径/衡量标准/价值形成一致看法 | **方案共创**：与客户共同制定实现路径和成功标准 | 共创："我们建议分三阶段提升OEE，您看这个阶段划分是否合理？" |

---

#### 1.3 四阶段推进策略

##### 阶段1：不清 → 常理

**目标**：帮助客户从"没意识到问题"到"认识到问题存在"

**策略**：
1. **提问唤醒**：用行业基准数据提问（"您知道行业平均OEE是多少吗？"）
2. **案例冲击**：分享同行业企业的成功案例（"某客户通过提升OEE 15%，每年节省300万"）
3. **痛点可视化**：帮助客户量化问题的影响（"如果OEE提升5%，您能多生产多少？"）

**话术模板**：
```
"XX总，我最近在服务[同类企业]时发现，他们通过[具体措施]将OEE从[现状]提升到了[目标]，
每年节省[金额]。不知道您这边的情况如何？"
```

---

##### 阶段2：常理 → 明确

**目标**：帮助客户从"大概知道"到"明确具体需求"

**策略**：
1. **量化澄清**：将模糊需求转化为可量化指标（"您说的'提升效率'具体是指？"）
2. **优先级排序**：帮助客户识别最紧迫的需求（"在您提到的3个需求中，哪个最紧迫？"）
3. **影响分析**：分析需求不满足的后果（"如果OEE不提升，对您的年度目标有什么影响？"）

**话术模板**：
```
"XX总，您刚才提到希望提升生产效率。为了帮您制定更精准的方案，
我想确认一下：您期望的提升幅度是多少？希望在多长时间内实现？"
```

---

##### 阶段3：明确 → 共识

**目标**：帮助客户从"个人需求"到"组织共识"

**策略**：
1. **方案共创**：与客户共同制定实现路径（"我们建议分三阶段实施，您看是否合理？"）
2. **成功标准共创**：与客户共同定义项目成功的衡量标准
3. **利益相关者对齐**：确保客户内部关键人都认同方案（"是否需要安排一次与[相关角色]的沟通？"）

**话术模板**：
```
"XX总，基于您的需求，我们初步拟定了这个解决方案。
我想邀请您一起评审：这个方案是否符合您的期望？
还有哪些需要调整的地方？"
```

| 错误 | 表现 | 后果 | 正确做法 |
|:----:|:------|:------|:--------|

### 一、信任五环® - 以客户为中心的销售拜访方法论

信任五环是指导销售拜访的核心框架，包含五个递进式沟通环节：

```
    ①拜访准备 → ②了解概念 → ③呈现优势 → ④获得承诺 → ⑤拜访评估
                                  ↑                        │
                                  └────── 反馈循环 ─────────┘
```

| 五环 | 核心任务 | 关键动作 | 产出 |
|:---:|---------|---------|------|
| **①拜访准备** | 研究目标角色，设计拜访路径 | SSO目标、行动承诺、PPP约见法 | 拜访计划+PPP剧本 |
| **②了解概念** | 建立专业形象，探索客户认知 | 四类提问、SPIN、黄金静默、心灵共振 | 客户认知地图 |
| **③呈现优势** | 将自身优势与客户概念精准链接 | 四季沟通术、BAF说服法、SPAR呈现、WSH差异模型 | 差异优势矩阵 |
| **④获得承诺** | 推动客户给出明确下一步承诺 | 行动承诺阶梯、LSC顾虑处理、GROW模型 | 行动承诺 |
| **⑤拜访评估** | 系统复盘、信任度判断、策略调整 | 七维评估清单、信任度诊断 | 拜访复盘报告 |

---

#### 第一环：拜访准备

**1.1 核心概念——"概念"**

> **概念 = 客户的认知与期望——实现什么、解决什么、避免什么**
> 是客户内心深处的感觉，并由此衍生的具体需求。

关键特征：
- 概念是"个人的"，不是公司的
- 概念是"不断变化的"，源于环境变化和自身感受变化
- 有些概念客户会直接说出来，很多需要我们去探索
- **只有符合客户概念的东西他才会花钱购买**

> "没有差距就没有购买。'现状'与'期望'认知之间的差距，就是客户改变的原动力。"

**1.2 客户需求的三个阶段**

| 阶段 | 思考方式 | 特征 |
|------|---------|------|
| **认识性思考** | 外界变化刺激 → 客观认知 → 产生改变想法 | 购买原动力 |
| **理性思考** | 形成行动标准/解决方案标准 | 发散性思考，考虑各种可能性 |
| **优选/聚敛** | 在符合标准的方案中做出最后选择 | 收敛决策 |

**1.3 行动承诺（Action Commitment）**

> **定义**：客户的具体动作，是推进项目进程的、需要客户投入时间和精力去做的具体事情。

**三个标准**：
1. **具体动作**：不能只是"后面我会联系你"，要有具体行为
2. **现实合理**：①客户职权范围内 ②能力范围内 ③销售流程现阶段应该做的
3. **源于客户概念和个人利益**：必须满足客户的认知与期望

**制定要点**：
- 仔细分析是否源于客户概念、满足客户个人利益
- 针对一次拜访设定**一个最佳行动承诺** + **一个最低行动承诺**
- 客户不给承诺时，可探索顾虑深层原因，或降低承诺标准

**1.4 PPP约见法（Purpose/Process/Payoff）**

> **定义**：客户眼中的约见理由，与客户概念和个人利益相关，在客户看来是重要和紧急的。

**六大作用**：让客户知道会谈真正目的；建立沟通预期；表示有备而来；缩短拜访时间提高效率；让客户有所准备；显得更专业增加信任。

**PPP三要素**：
- **Purpose（目的）**：我们为什么要见面
- **Process（过程）**：我们将如何进行
- **Payoff（收益）**：这些时间对彼此有什么好处

**示例**："XX主任，今天希望针对XX项目就您关心的实施风险控制做个交流（P目的）。主要想听听您认为的风险有哪些，如何才能有效规避，以及需要我们怎么配合（P过程）。这样既能让您有效控制风险，也能让我们有所准备（P收益）。"

#### 第二环：了解概念

**2.1 建立信任与好感的三个方法**

**① 自我介绍**：站在客户角度，讲自己的经验、成功案例可能给他带来什么帮助；注意措辞——说客户可能存在的"目标"而非"问题"（没人愿意被直接指出问题）。

**② 暖场类问题**：留心观察"此时此地"（办公室装饰、摆件）；聊客户工作成就、关心话题；必须是开放式问题，视人而定。

**③ 心灵共振**：站在客户角度，考虑客户当下最关注的问题。金句："我在来的路上还在想，您最关心什么呢？"
- **问题清单法**：把客户可能关心的问题写在一张清单上交给客户，极具杀伤力

**2.2 四类关键问题**

| 问题类型 | 功能 | 要点 | 示例 |
|---------|------|------|------|
| **确认类问题** | 核实信息、对等理解 | 封闭式为主 | "我理解刚才您说的是……是吗" |
| **信息类问题** | 了解现状、挖掘事实 | 开放式，先求同后存异 | 针对具体业务/情境从现状问起 |
| **态度类问题** | 探索感性感受、动机 | 四种问法（干什么/怎么办/哪一个/描述） | 切忌臆断客户态度 |
| **SPIN提问** | 系统挖掘深层需求 | S现状→P困难→I影响→N价值 | 只有对**个人**产生影响客户才会行动 |

**SPIN详解**：
- **S（Situation）**：询问客户现状
- **P（Problem）**：询问客户困难
- **I（Implication）**：让客户认识到困难会带来什么潜在影响
- **N（Need-Payoff）**：问题解决之后会有什么价值
- **难点**：只有业务问题对**个人**产生了影响（压力、痛苦或更大利益），客户才会行动
- **注意**：中国人更喜欢讨论目标而非痛苦

**2.3 倾听五层次**

| 层次 | 名称 | 特征 |
|------|------|------|
| 一 | 听而不闻 | 一心只想自己要说的话 |
| 二 | 心不在焉 | "嗯……是……对对对"虚应 |
| 三 | 听我想听 | 只选择自己想听的话 |
| 四 | 专注地听 | 每句话都进大脑，但不一定真正理解 |
| 五 | **将心比心** | 放下固有思维，随对方思维而动（最高层次） |

**2.4 六种回应方式**

| 类型 | 用法 | 示例 |
|------|------|------|
| **追踪式** | 针对关键信息深入发掘 | "你能讲具体点儿吗""能举个例子吗" |
| **征求看法式** | 探索对方想法和态度 | "您是怎么想的""您怎么看" |
| **极限式** | 明确最关心、最顾虑的问题 | 问"最"的问题 |
| **魔法师式** | 移开思维障碍，引导发散思考 | "如果数据可以及时统计，您希望成本精细到什么程度" |
| **肢体语言** | 坐姿前倾、目光看眉眼与嘴之间、微笑、缓缓点头（1秒1次，3-5次） | — |
| **沉默是金** | 提问后沉默3-4秒→客户第一次回答→再沉默3-4秒→客户细化回答 | "点燃二踢脚" |

> **"能不能问出专业的问题"是衡量销售是否专业的标准，而非"能不能讲专业的方案"。**

**2.5 肢体语言关键数据**

面对面谈话中：身体语言55% + 语调38% + 语言内容仅7%
- 抚摸下巴 = 在思考你的话
- 交叉双臂/仰面靠椅 = 可能提出反对
- 双臂舒展/身体前倾 = 可能给出肯定

#### 第三环：呈现优势

**3.1 购买决策逻辑**

> 客户根据自己现状与期望的**差距**决定**买不买**；根据各供应商之间的**差异**决定**买谁的**。

- **理性差异**：参与客户理性分析、比较方案产品服务的过程
- **感性差异**：无法客观描述，但往往是决策关键，尤其在理性差异不大时

> "产品只有特点，没有优点和缺点。客户针对他的概念看到优势和劣势，抛开概念就没有优劣势可言。"

**3.2 客户采购三阶段**

| 阶段 | 类型 | 时间精力占比 | 识别信号 |
|------|------|:-----------:|---------|
| **概念形成** | 认识过程 | 60% | 客户问"你们这个是怎么实现的" |
| **标准制定** | 歧义过程 | 30% | 竞争显现、对细节交流多、演示时长增加 |
| **做出决定** | 敛聚过程 | 10% | 双方回顾工作、关键人出面参与 |

> **策略**：销售高手在客户明确需求之前下手，把工作做在前面，风险控制到最低。

**3.3 概念铁球（核心隐喻）**

客户的概念躲在空心铁球内部，我们的优势在铁球外面。如果直接把优势关联概念，中间隔着一层厚铁壳——人们潜意识抗拒外来强加的观点。

> **正确做法**：从了解客户概念开始 → 和客户共同制定标准 → 将方案与概念标准结合 → 合作制定双方共同拥有的方案 → 使双方都有意愿做出行动承诺。

**3.4 四季沟通术**

| 季节 | 含义 | 做什么 | 听/说 |
|------|------|--------|:-----:|
| **春播** | 了解客户概念 | 发现/播下种子 | 概念—听 |
| **夏耕** | 征求实现标准 | 引导客户思考需要什么特性 | 方案—听 |
| **秋收** | 呈现差异优势 | 客户形成认识后做出决定 | 方案—说 |
| **冬藏** | 确认愿景 | 让客户看到概念如何被满足 | 概念—说 |

- **春夏**：提问、倾听、探索、讨论
- **秋冬**：呈现、讲述、解释、确认
- **本末关系**：销售的"本"是客户的概念，"末"是方案

**3.5 BAF说服法（反转FAB）**

| 传统FAB顺序 | ✅ 正确BAF顺序 |
|------------|--------------|
| F（功能）→ A（特点）→ B（利益） | **B（利益）→ A（特点）→ F（功能）** |

FAB从产品/销售角度出发，BAF从客户角度出发——先谈客户想得到什么利益B，再谈需要什么特点A才能实现，最后才讲产品有什么F可以满足。

**3.6 WSH差异优势模型**

> 决策基于差异：**因相同接受你，因不同选择你**

| 字母 | 含义 | 说明 |
|------|------|------|
| **W** | What——差异是什么？ | 明确你的独特优势点 |
| **S** | So What——这意味着什么？ | 该差异对客户的实际意义 |
| **H** | How to prove——如何证明？ | 用证据支撑差异优势 |

**独特差异优势五要素**：①有什么优势 ②针对谁的优势 ③谁眼中的优势 ④那又怎么样 ⑤何以证明

> 证据提交要像法庭辩论——在逻辑最薄弱环节适时亮出证据，而非一上来就丢资料。

**3.7 SPAR呈现法（感性决策路径）**

| 要素 | 含义 |
|------|------|
| **S**（Situation） | 状况，在什么情况下 |
| **P**（Person/Role） | 某个人/角色 |
| **A**（Action） | 做什么事，有什么动作 |
| **R**（Result） | 效果、影响和感受（对业务的影响+对个人的影响） |

**三大要求**：场景化（冲击右脑）、具体化（具体到每个人每个动作）、感性化（突出感觉感受）。

**表达原则**：每过五分钟要回到客户信息和态度的确认探讨；持续说超过七分钟很可能没顾及客户想法；最好的沟通是把说的话变成问题再抛回去。

**3.8 FABE理性分析路径**

| 要素 | 含义 |
|------|------|
| **F**（Feature） | 产品/方案的特征属性 |
| **A**（Advantage） | 该特征带来的优点 |
| **B**（Benefit） | 给客户带来的具体好处 |
| **E**（Evidence） | 支撑上述内容的证明材料 |

> **实操**：FABE理性分析 + SPAR感性呈现 双管齐下。需制定《差异优势表》，对比自身产品、竞争者与客户认知期望。

**3.9 概念树/目标树/优势树（知识积累工具）**

1. 列出销售中可能遇到的所有角色（董事长→总经理→副总→处长→科长→员工），分别建立档案
2. 每次拜访中了解到的概念、目标、优势，记录在对应岗位/角色下
3. 长期积累后分类汇总，变成可以提问的好问题和可以说的话术

#### 第四环：获得承诺

**4.1 顾虑 vs 反对（核心区分）**

| | 反对 | 顾虑 |
|--|------|------|
| **性质** | 理性思考后形成 | 感性的，只是担心 |
| **表现** | 能具体指出来哪里不行 | 不敢和不愿意，无形的感觉 |
| **根源** | 针对具体事实 | 源于态度和看法，内心深处 |
| **转化** | 顾虑受外界压力后发展出具体证据 → 形成反对意见 | — |

**客户顾虑的是或许会失去的东西**：权力、控制、被认同、领导信任、被赞赏、被尊重、安全感、自由支配时间等。

> **永远不要假设或臆断客户真正的顾虑是什么**。直接问是最好的办法。

**4.2 LSC太极推手（处理反对）**

| 步骤 | 含义 | 做法 |
|------|------|------|
| **L**（Listen） | 专注倾听 | 让客户充分表达疑虑，不打断 |
| **S**（Share） | 理解并分享感受 | "我很理解你的感受"（心灵共振） |
| **C**（Clarify） | 探索并澄清 | "能告诉我是什么原因让你这么想的吗" |

核心策略：不直接回复客户问题（"不顶"），不直接出手接招或回招。可引用第三方实例或自己心路历程找共通点。

**LSC-CC顾虑处理完整模型**：
| 步骤 | 操作 |
|------|------|
| **L** | 完整听取客户顾虑 |
| **S** | 与客户感受产生共鸣 |
| **C** | 明确具体顾虑点 |
| **C**（Solution） | 提供解决方案 |
| **C**（Confirm） | 确认客户是否接受 |

**4.3 GROW模型（获得承诺的框架）**

| 要素 | 含义 |
|------|------|
| **G**（Goal） | 了解客户的目标是什么 |
| **R**（Reality） | 了解现状怎么样 |
| **O**（Options） | 询问对方对差距的看法 |
| **W**（Will） | 接下来将采取什么行动 |

**4.4 SPI九格愿景构建模型**

- **横向**：开放式、控制式、确认式三类问题
- **纵向**：诊断、探究、构建三个步骤
- 比喻：九格是一把枪，子弹就是专业知识

**4.5 CCS客户中心销售法**

将SPI的"痛"转化为"目标"：构建目标 → 诊断差距 → 确认差距 → 构建做法 → 诊断价值 → 确认价值

> 现在客户越来越成熟，谁哪儿痛谁自己知道，没必要再往痛处捅。

**4.6 销售漏斗六阶段**

发现商机 → 确认意向 → 引导立项 → 赢得认可 → 商务谈判 → 完成销售

关键原则：
- 每个阶段要明确面对哪个角色，和谁一起完成
- 销售成功靠内部人支持，用内部人支持消除不支持（固强制弱）
- **成交由客户的多个行动承诺构成**，客户连续上几个台阶，销售才能成功

#### 第五环：拜访评估

**七维评估清单**：
1. 是否制定了有效的约见理由
2. 准备了什么好问题
3. 呈现之前是否了解客户的概念
4. 在使用"沉默是金"上有多成功
5. 客户给出了哪些行动承诺，与最佳行动承诺相关性有多大
6. 这些行动承诺对推进销售有多大帮助
7. 我们帮客户解决了哪些顾虑，客户还有什么顾虑

**提问禁忌**：
- ❌ 少用"为什么"（让人感觉被质问和挑战）
- ❌ 不要问"都知道"的问题（互联网时代无价值）
- ❌ 不要一上来就问预算、项目计划、竞争对手情况

#### 🔍 识局篇（看清形势）

**第一问：客户究竟要什么？**

| 分析维度 | 关键问题 |
|---------|---------|
| 需求范畴 | 需求范畴和目标清晰吗？ |
| 认知归属 | 这是客户的认知还是我的认知？ |
| 选择空间 | 客户还有选择的其他方案吗？ |
| 环境变化 | 识别客户内外部环境的变化 |

**客户反应四类**：
| 类型 | 特征 | 策略 |
|------|------|------|
| **如虎添翼型（Growth）** | 现状不差，但愿意追逐梦想 | 展示增值价值，描绘愿景 |
| **亡羊补牢型（Trouble）** | 出了问题，急于补救 | 直击痛点，快速响应 |
| **两厢情愿型（Need）** | 有明确需求，理性选择 | 匹配需求，差异化竞争 |
| **锦上添花型（Opportunity）** | 有机会但紧迫度低 | 培育需求，长期经营 |

**SSO（单一销售目标）检查清单**：
- 跟我有关系吗？
- 清晰具体吗？
- 有预算吗？
- 有期限吗？
- 有竞争吗？

> "没有差异，就没有采购。" "销售目标是'定盘星'，一切行动围绕目标展开。"

**第二问：我的位置在哪里？**

**形势三维结构**：销售进程 × 客户紧迫程度 × 竞争形势

| 分析维度 | 关键问题 |
|---------|---------|
| 阶段判断 | 项目到了什么阶段？ |
| 踩刹车 | 什么时候该踩刹车？ |
| 漏斗误导 | 销售漏斗是否会误导判断？ |
| 紧迫程度 | 客户的紧迫程度怎么样？ |
| 竞争态势 | 我们在竞争中处于什么位置？ |
| 项目温度 | 如何感知项目温度？ |

**第三问：如何识别关键人？**

**"帅马车炮"角色模型（象棋类比）**：

| 象棋角色 | 对应销售角色 | 特征 | 关注点 |
|---------|------------|------|-------|
| **帅** | EB（最终决策者） | 一锤定音 | 投资回报、组织影响 |
| **马** | UB（应用选型者） | 灵活走位 | 工作方式、绩效影响 |
| **车** | TB（技术选型者） | 直来直去 | 技术标准、合规安全 |

**第四问：客户如何评价我？**

| 分析维度 | 关键问题 |

**态度标识体系（十级分类）**：

| 标识 | 含义 | 等级 |
|------|------|:---:|
| **G+5** | 热情拥护，坚定支持，不选我们就放弃 | 最高 |
| **G+4** | 大力支持，敢于当众抵制对手 | 高 |
| **G+3** | 支持，明确当众表示会支持我们 | 中高 |

**旗与雷标注法**（第四问核心工具）：

> 在策略分析表上，对每个关键角色标注"旗"（优势因素）和"雷"（风险/威胁因素）。

| 标注 | 含义 | 示例 | 策略 |
|:---:|------|------|------|
| **🚩 旗** | 我们在这个角色上的优势因素 | "张总对我们行业的案例很认可" | 保持、强化、不要把旗变雷 |
| **💣 雷** | 我们在这个角色上的风险/威胁 | "李工曾经用过竞品，印象不错" | 排雷优先——忽视的雷一定会爆 |

**旗与雷的转化规律**：
- **旗可以变成雷**：过度依赖某一面旗（"张总全力支持我们"），忽视其他关键人，最终张总因内部压力反转 → 旗变雷
- **雷可以排除**：识别到雷后，提前采取措施（安排技术交流消除李工对竞品的偏好）
- **"最危险的雷是你看不到的雷"**：未被识别的影响者、未暴露的竞品关系

**角色参与程度曲线**（第五问核心洞察）：

> 不同角色在采购不同阶段的参与度和影响力是**动态变化**的，而非固定不变。

```
参与程度
  ↑
高│     UB╲        ╱EB
  │        ╲      ╱
中│     TB ╲    ╱  TB
  │         ╲  ╱
低│         EB╲╱        UB
  └──────────────────────→ 采购阶段
    需求识别  方案评估  决策选择  实施

UB曲线：高→中→中下→中下（前期影响大，后期被边缘化）
TB曲线：中→高→中→中下（方案评估期是TB的主场）
EB曲线：低→中下→高→高（越到决策阶段影响力越大）
```

**各阶段影响力排名**：
| 采购阶段 | 影响力排序 | 销售动作 |
|:-------|:---------|---------|
| **需求识别** | UB > TB > EB | 重点争取UB，了解其痛点 |
| **方案评估** | TB > UB > EB | 技术方案征服TB，TB说"行"才可能往下走 |
| **决策选择** | EB > TB > UB | 此时才到EB的主场，前面的工作都是为了这一刻 |
| **实施落地** | UB > EB > TB | 实施效果由UB口碑决定，影响续费和增购 |

**实战启示**：
- **不要在错误的时间找错误的人**：项目初期就去找EB聊价格 → 浪费EB时间且留下负面印象
- **TB是"守门员"，UB是"敲门砖"**：先让UB认可价值，再让TB通过技术关，最后让EB拍板
- **很多人忽视EB在早期的影响力**：虽然EB早期不直接参与，但他的一句话可以随时改变项目走向

**接近客户高层的四步法**（第七问核心策略）：

```
第一步：识别真正的决策者
  ├─ 不是头衔最大的人，而是对预算有最终支配权的人
  └─ 提防"影子决策者"——不在组织架构图上但实际说了算的人

第二步：克服约见阻挡
  ├─ 分析阻挡者的动机（秘书/助理/中层）→ 不是敌人，是通道
  └─ 让阻挡者觉得"让你见老板对他有好处"

第三步：准备有效约见理由
  ├─ 必须明确具体 → "我想和张总聊聊行业最新趋势对贵司的影响"
  ├─ 让客户感觉有价值 → 不是"我想推销"，而是"我有对你有用的信息"
  └─ 一句话说清：为什么见我、对我有什么好处

第四步：见面后用价值留住关注
  └─ 一次见面的价值决定下次见面是否可能
```

**九类可调用资源**（第八问完整清单）：

| # | 资源类型 | 作用 | 使用时机 | 注意事项 |
|:--:|---------|------|---------|---------|
| 1 | **专业顾问** | 弥补售前技术短板 | 方案评估阶段 | 提前briefing，确保与销售口径一致 |

| 3 | **销售的上司** | 对等对话，表达重视 | 高层互访 | 上司出场前务必备齐所有背景信息 |
| 4 | **公司的高管** | 展示公司战略承诺 | 关键决策点 | 高管是"核武器"，不轻易用，一用必见效 |
| 5 | **合作伙伴** | 补充方案拼图 | 方案设计阶段 | 确保合作伙伴可控，不反水 |

| 9 | **竞争对手** | 借力打力 | 客户摇摆时 | ⚠️ 双刃剑——利用对手的错误而非主动攻击 |

**资源使用四原则**：
1. **有明确目标**：申请资源前问自己"这个资源要解决什么问题？"
2. **针对具体问题**：不能"约高管聊聊"，必须"针对XX问题请高管表态"
3. **不求最好重在适用**：大专家解决小问题 = 浪费且不合适
4. **关注资源的"赢"**：每个被你调用的资源也需要获得成就感

**资源申请三原则**（说服内部审批）：
1. 明确资源用途和预期效果（"安排一次高管访问，预期可加速EB决策2周"）
2. 说明不用资源的风险和损失（"如果不做POC，TB会被竞品说服"）
3. 提供资源使用的具体方案（时间/地点/参与人/议程/产出）

**第九问：面对竞争怎么办？**

| 竞争态势 | 策略 | 关键动作 |
|---------|------|---------|
| **我方领先** | 巩固策略 | 强化优势、加速推进 |
| **双方持平** | 差异化策略 | 找到独特价值、不可替代 |
| **我方落后** | 反超/侧翼策略 | 找差距补短板、绕开正面 |
| **对手根深蒂固** | 渗透策略 | 从边缘入手、逐步突破 |
| **陷入价格战** | 价值策略 | 避免大幅杀价、回归客户价值 |

**关注客户而非对手**（第九问第一原则）：
> "不要被竞争对手的节奏带乱。竞争的焦点是客户心中的认知，不是对手在做什么。"

- 客户问"你们和XX比有什么优势" → 不要直接比较对手，先反问"您怎么评价XX的方案？"
- 过分关注对手 = 承认自己不如对手。你的优势是由客户来定义的
- **"只有客户认知的优势才是优势，只有客户认知的差异才是差异"**

**无效降价的真相**：

> "大幅杀价往往无效。降价只能解决'价格'问题，解决不了'价值'问题。如果客户觉得你的东西不值钱，你降价他更觉得不值钱。"

| 降价策略 | 结果 | 为什么 |
|:-------|------|------|
| 贸然降价 | 客户怀疑"之前想多赚我多少？" | 破坏了信任基础 |
| 降价对标对手 | 陷入对方的价格锚定 | 你在对手的规则里玩 |
| 让价值与价格对等 | 客户觉得"值这个价" | 价值认知决定价格接受度 |

> "不是在价格上让步，而是让客户认识到你的价值与他付出的价格对等。"

**竞争对手根深蒂固时的策略**：
1. 不要在对手的强项上硬拼 → 找到"鸡蛋上的裂缝"
2. 在客户机构里与更多的人接触 → 找到未被对手覆盖的需求盲区
3. 关注变化带来的机会 → 管理层变动、业绩压力、新战略方向
4. 先让客户认识到"你也能达到对手的水平"，再寻找差异

**应对客户邀标的决策框架**：

| 情况 | 决策 | 理由 |
|------|:--:|------|
| 已充分了解客户需求，有赢单把握 | ✅ 参与 | 投标是收割，不是探路 |
| 客户需求不明，只是"试试看" | ❌ 不去 | "没有充分了解就不要去投标" |
| 不去投标但不想得罪客户 | 📝 礼貌回复 | 说明"目前对贵司需求了解不够深入，待充分沟通后再参与" |
| 已被邀请但明显是陪标 | ⚖️ 权衡 | 判断是否值得用一次投标换取未来机会

### 四、客户拜访全流程

#### 4.1 拜访前准备

**商务准备**：客户背景资料、行业知识储备、竞争对手信息、相关案例准备

**销售准备**：单一销售目标(SSO)、问题清单准备、客户应对话术、成功案例素材

**拜访函（PPP）**：
- **Purpose（目的）**：我们为什么要见面？
- **Process（过程）**：交流将如何进行？
- **Payoff（收益）**：对彼此有何好处？

#### 4.2 拜访中沟通

**提问技巧**：
- 信息类问题：了解现状和背景
- 态度类问题：了解期望和想法
- 行动类问题：了解计划和行动
- 确认类问题：确认共识和承诺

**沟通节奏**：开场建立信任 → 探询了解需求 → 呈现解决方案 → 处理异议顾虑 → 获取行动承诺

#### 4.3 拜访后跟进

1. 回顾沟通过程
2. 确认双方共识
3. 总结阶段成果
4. 明确下步计划
5. 推进销售进程

## 🔥 量化模型体系（V2.0增强）

### 一、SSO（单一销售目标）评估模型

**SSO完整性评分公式**：
```
SSO完整性得分 = Σ(SSO要素得分 × 权重) × 100%
```

| SSO要素 | 判断标准 | 分值 | 权重 |
|---------|---------|------|------|
| **相关性** | 与客户核心业务目标直接相关 | 0-25分 | 0.20 |
| **清晰度** | 目标具体、可量化、可描述 | 0-20分 | 0.15 |
| **预算** | 有明确预算或预算范围 | 0-20分 | 0.20 |
| **期限** | 有明确的决策时间节点 | 0-15分 | 0.15 |
| **竞争** | 竞争态势清晰、策略明确 | 0-10分 | 0.15 |
| **可及性** | 我们有能力满足需求 | 0-10分 | 0.15 |

| 健康等级 | SSO得分 | 含义 | 行动 |
|---------|---------|------|------|
| **健康** | ≥80% | SSO完整清晰 | 按计划推进 |
| **关注** | 60-79% | SSO基本完整 | 补充缺失要素 |
| **预警** | 40-59% | SSO有重大缺失 | 暂停推进，先补SSO |
| **危险** | <40% | SSO严重缺失 | 不建议投入资源 |

---

### 二、赢单概率计算模型（WPV）

**赢单概率计算公式**：
```
WPV = Σ(维度得分 × 权重) × 阶段调整系数 × 竞争调整系数
```

| 评估维度 | 权重 | 评分标准(0-10分) |
|---------|------|-----------------|
| **需求匹配度** | 0.20 | 我方方案与客户需求匹配程度 |
| **决策链完整度** | 0.20 | EB/UB/TB/Coach各层级覆盖 |
| **关系紧密度** | 0.20 | 与关键决策人的信任程度 |
| **竞争位置** | 0.15 | 相比竞争对手的优劣 |
| **方案差异化** | 0.15 | 独特价值主张的强度 |
| **资源投入度** | 0.10 | 我方投入资源的充分程度 |

| 赢单等级 | WPV分值 | 含义 | 策略 |
|---------|---------|------|------|
| **高概率** | ≥75% | 优势明显 | 加速推进，减少变数 |
| **中概率** | 50-74% | 需努力争取 | 制定赢单策略，补短板 |
| **低概率** | 25-49% | 困难较大 | 评估ROI，考虑资源转移 |
| **极低概率** | <25% | 机会渺茫 | 减少投入，准备撤退 |

---

### 三、角色影响力评分模型（RIS）

```
RIS = 职位影响力 × 组织位置系数 × 个人决策权 × 关系紧密度
```

| 角色 | 职位影响力基准 | 核心关注点 |
|------|--------------|-----------|
| EB | 40分 | 投资回报、组织影响 |
| UB | 25分 | 工作方式、绩效影响 |
| TB | 20分 | 技术标准、合规安全 |
| Coach | 15分 | 帮助制定策略 |

**决策链健康度**：
```
决策链健康度 = Σ(各角色影响力 × 态度系数) / Σ(各角色影响力) × 100%
态度系数：强烈支持=1.0 / 支持=0.8 / 中立=0.5 / 反对=0.2 / 强烈反对=0.0
```

| 健康等级 | 健康度分值 | 含义 | 策略 |
|---------|-----------|------|------|
| **优秀** | ≥80% | 支持占主导 | 加速推进 |
| **良好** | 60-79% | 支持略占优 | 补强薄弱环节 |
| **一般** | 40-59% | 支持与反对持平 | 重点突破反对者 |
| **较差** | <40% | 反对占主导 | 重新评估策略 |

---

### 四、资源投入产出模型（RIO）

**预期回报** = 商机金额 × 赢单概率 × 利润率贡献 × 战略价值系数

| 战略价值类型 | 系数 |
|------------|------|
| 标杆客户 | 1.3 |
| 战略新领域 | 1.2 |
| 大客户维稳 | 1.1 |
| 普通商业机会 | 1.0 |
| 清库存/尾单 | 0.8 |

| ROI区间 | 决策 |
|--------|------|
| ≥3.0 | **优先投入** |
| 2.0-3.0 | **正常投入** |
| 1.5-2.0 | **谨慎投入** |
| <1.5 | **不建议投入** |

---

### 五、拜访效率评估模型（VES）

```
VES = (信息获取分×0.3 + 关系推进分×0.25 + 问题解决分×0.25 + 承诺获取分×0.2) × 100
```

| 等级 | VES得分 | 改进建议 |
|------|--------|---------|
| **卓越** | ≥85分 | 总结复用 |
| **良好** | 70-84分 | 针对提升 |
| **一般** | 55-69分 | 专项训练 |
| **欠佳** | <55分 | 全面复盘 |

## 控单力提升专项（v10.0新增）

> 核心思想：AR不是销售员，是导演。客户选型不是自然发生的——是被人设计出来的。

### 一、AR导演视角：让客户按你的剧本演

**核心认知转换：**

```
传统销售思维                        AR导演思维
─────────────────                  ─────────────────
"客户要什么我就给什么"              "我引导客户要什么"
"客户问我回答"                      "我先问客户再给答案"
"客户定节奏"                        "我定节奏和里程碑"
"我在帮客户选型"                   "客户按我的剧本走"
"见招拆招"                          "提前设计好所有牌"
```

**导演三件事：**
1. **定剧本**：设计客户选型流程的里程碑和节奏（什么时间做什么决策）
2. **选演员**：识别决策链每个角色，给每个角色安排"戏份"
3. **控场**：管理竞争对手的出场时间和位置，让自己始终在C位

### 二、客户采购阶段 vs 我方控单过程映射

这是控单力提升的核心工具——理解客户在哪个阶段，我方必须做什么：

| 阶段 | 客户采购流程 | 我方控单动作 | 关键产出 | 里程碑 |
|:---:|:----------|:-----------|:--------|:------|
| **1** | 问题识别 | 需求唤醒 + 痛点引导 | 客户认知到问题严重性 | 客户认可痛点 |
| **2** | 方案探索 | 方案引导 + 价值主张 | 客户接受我方解决思路 | 方案方向确定 |
| **3** | 方案设计 | 需求锁定 + 优势构建 | 我方需求被写入标书 | 需求文档确认 |
| **4** | 评估比较 | 差异化呈现 + 竞品压制 | 客户认可我方差异优势 | POC/演示通过 |
| **5** | 决策选择 | 风险消除 + 内部动员 | 决策链多数支持我方 | 内部推荐形成 |
| **6** | 实施交接 | 商务谈判 + 交付承诺 | 合同签署 | 签约 |

**每阶段核心判断：**
- 客户当前在哪个阶段？→ 匹配对应动作，不跳阶、不滞后
- 谁在帮我们推进到这个阶段的下一阶段？→ Coach的能力验证
- 竞争对手在哪个阶段做了什么？→ 情报判断

### 三、隐性需求挖掘（医生式追问法）

**核心问题**：客户说"我需要XX功能"——但这往往不是真实需求。

**医生式追问三连：**
```
客户："我需要一个自动化报表功能"
销售追问1："您要这个功能是解决什么痛点？"
  → 客户："我现在手动做报表要3天"
销售追问2："是什么导致了要做3天？"
  → 客户："数据要从3个系统手工汇总"
销售追问3："如果能解决这个问题，对您个人意味着什么？"
  → 客户："我每周能省出2天做分析，老板会更认可我"
```

**隐藏动机金字塔**：
```
表面需求                  → "我要XX功能"
  ↓ 第一层追问（痛点）
业务痛点                  → "手动做报表要3天"
  ↓ 第二层追问（根因）
根本原因                  → "3个系统数据不互通"
  ↓ 第三层追问（个人动机）
个人赢                    → "老板认可我，晋升有资本"
```

### 四、决策链干系人管理工具

**决策角色识别（升级版）：**

| 角色 | 核心关注 | 我方策略 | 验证方式 |
|:----|:--------|:--------|:--------|
| **EB**（最终决策者） | ROI/战略价值/风险 | 高层对话+案例背书 | 能否直接约见？约见后态度？ |
| **UB**（应用选型者） | 使用效果/工作效率 | 场景演示+价值量化 | 是否主动支持我方？ |
| **TB**（技术选型者） | 技术标准/合规 | 技术方案+安全白皮书 | 需求文档是否对我不利？ |
| **Coach**（教练） | 项目成功/个人信任 | 信息互换+策略对齐 | 能否提供对手情报？ |

**干系人投票策略**：
- 目标是：让决策链上多数干系人支持我方
- 策略：从下往上，逐层攻克
- 话术：对每个干系人，讲"这对你的KPI意味着什么"

### 一、开篇核心隐喻 — 策略销售的底层认知

#### 1.1 策略 vs 战术：《孙子兵法》的销售启示

| 维度 | 策略（谋） | 战术（术） |
|:----:|---------|---------|
| **定义** | 分析形势、制定作战计划、排兵布阵的艺术 | 骑马射箭、舞刀弄枪的基本技巧 |
| **解决** | 为什么做（Why） | 怎么做（How） |
| **比喻** | 赤壁之战中诸葛亮的"借东风" | 赵云、黄盖的冲锋陷阵 |
| **销售对应** | 在正确的时间、地点、见正确的人、讲正确的话 | 拜访沟通、演讲谈判、产品知识 |

**核心认知**：
- 方向错了，跑得越快，离目标越远。**刀枪保命，策略制胜！**
- 策略与战术是《周易》中的"阴"与"阳"，一阴一阳、一静一动，互相推演转化
- 《孙子兵法·军形第四》："胜兵先胜而后求战" — 销售应该是"先胜而后求战"，而非"先战而后求胜"
- 《孙子兵法·谋攻第三》："知彼知己者，百战不殆" — "彼"不是竞争对手，而是客户。应关注竞争对手的行动在客户身上产生的影响

#### 1.2 五大销售误导（公众对销售的错误认知）

| 误导 | 表现 | 真相 |
|:---:|------|------|
| **忽悠** | 滔滔不绝介绍产品功能特点，试图"创造需求" | 人们喜欢购买，讨厌被推销 — 不要剥夺客户的选择权 |
| **谋略** | 为达目的不择手段，用尽心机搞定客户 | 客户会问："你们就是用这样的手段来算计我们的？" |
| **勤快** | 提高拜访量、扩大漏斗基数来提高成功概率 | 盲目追求数量，无法深入分析每个客户的购买原因 |
| **自信** | "我是最棒的" — 过度以自我为中心 | 失去与客户的有效互动，不去倾听客户的真实想法 |
| **控制** | 给客户建立选型标准，消灭反对意见 | 假设"客户没有想法、是愚蠢的"，实际上客户早就不无知了 |

> **根源**：以上五种误导都建立在错误的假设之上。正确的假设是 — 客户有自己的"概念"（认知与期望），销售的任务是帮助客户实现他的概念，而非把自己的概念强加给客户。

#### 1.3 黑屋子隐喻 — 客户的购买决策过程

```
① 打开灯 → 看清所处形势
② 想出屋 → 挖掘各种可能
③ 找钥匙 → 选择最佳方案
```

这正是心理学家吉尔福德《人类智能的本性》中的人类意识决策过程：
- **理解（认识过程）**：客户意识到问题存在
- **产生（歧异过程）**：客户考虑各种可能的解决方案
- **优选（聚敛过程）**：客户在可选方案中做最终选择

> **关键发现**：传统销售过程是"逆着"这个人类意识决策过程的 — 客户刚说"要配钥匙"，我们就开始介绍产品。正确的做法是先帮客户"打开灯"和"想出屋"，再谈"配钥匙"。

#### 1.4 鸡过马路隐喻 — 客户的"概念"

> "鸡为什么过马路？答案只有鸡自己知道。如果要想知道真正的答案，那只能'去问鸡'！"

**客户的概念** = 客户自己内心知道的、购买所要达到的目的、愿景和感觉。
- 概念是个人的，不是公司的
- 概念是不断变化的
- 概念有时连客户自己也无法清晰表达
- **只有符合客户概念的东西，他才会花钱购买**

> "当我们不知道客户'概念'的时候，任何的推销都是在乱枪打鸟。"

#### 1.5 千万大单失败案例复盘（完整叙事）

> 开篇案例：价值数千万元的信息化项目，经过一年多精心运作 — 需求调研、方案讲解、样板参观、高层表态支持、标书由我方起草、"一切尽在掌握"。最终却被一家从未调研、从未参观、客户从未看过方案的外国公司以最高价中标。为什么？

**失败根因分析**：
1. 自以为"一切尽在掌握"，陶醉于表面支持
2. 忽略了评标现场出现的"陌生面孔"（上级控股公司代表）
3. 没意识到支持者可能在关键时刻不表态
4. "客户是感性决策，寻找理性依据加以证明"
5. 赢单的外国公司把"策略销售"做进了他们的销售管理系统

**教训**：从天堂到地狱，只有半步之遥。**早知道的坏消息就是好消息** — 项目出现问题的环节，往往就是我们把假设当成结论的悬疑环节。

### 二、成功销售的必备素质（四维能力模型）

> 来源：《赢单九问》第465-468页

| 维度 | 核心能力 | 关键要求 |
|:---:|---------|---------|
| **自我管理** | 心态调适、时间管理、适应变化 | 能"以爱人之心服务客户"，保持正直诚恳 |

| **周详的分析策划能力** | 形势分析、策略制定、资源协调 | 销售就是战役指挥官："运筹帷幄之中，决胜千里之外" |

### 三、销售之道 — 九大哲学原则（跋：销售这点事）

> 来源：《赢单九问》第469-486页跋文。策略销售闭环系统的最终哲学归依。

#### 3.1 忠恕而已
> "夫子之道，忠恕而已矣。"（《论语·里仁》）

| 原则 | 含义 | 销售应用 |
|:---:|------|---------|
| **忠** | 忠于自己内心的感受 | 一个订单能不能成交、何时成交，答案就在自己心里，感觉骗不了人 |
| **恕** | 揣度、体谅他人的心思 | 知道客户的心思 — 客户要什么才决定我们卖什么 |

**实践法则**：白天问客户的想法，晚上问自己的感受。问得越多，把握得越清楚。

**核心警示**：
- 项目关系重大时，人不愿出局 → 过早进入心理防御状态 → 把假设当成结论 → 欺骗自己说风险不会发生
- **墨菲定律**：担心出问题的地方一定会出问题

#### 3.2 谋定后动，知止有得

> "夫未战而庙算胜者，得算多也。"（《孙子兵法·始计篇》）

- 多数销售匆匆忙忙穿梭于客户之间，熙熙攘攘忙忙碌碌 — 却很少停下来分析项目整体状况
- **孔子**："无欲速，无见小利。欲速，则不达；见小利，则大事不成。"
- 销售就是司机，不能一味踩油门，还要看路 — 每时每刻知道自己在哪、要去哪

**定期项目回顾清单**：
- 存在哪些风险？
- 哪些重要信息不确定或遗漏？
- 哪些关键人员还没有接触？
- 哪些人明显反对我们？
- 出现了哪些新面孔？
- 关键人物位置是否变化？
- 决策周期和程序有没有新变化？

#### 3.3 各谋其事，各司其职

- 有权的人可能不想买，想买的人可能没权；把关的人不着急，着急的人不把关
- **"别卖给无法购买的人！"** — 准确识别决策影响者角色、发掘全部决策影响者至关重要

#### 3.4 察其所由，观其所以

- 客户的思维是"管状的" — 只关心自己的业务，专心解决自己的问题
- **G/T/EK/OC** 反馈形态的根源：使用者认为有价值→积极购买；采购者觉得是负担→应付；决策者看不到明显受益→不急
- 客户对项目的态度 ≠ 对供应商的态度
- 得到认同、拥有权力、受到尊重、获得赞扬 — 谁能满足客户的心理诉求，就支持谁

#### 3.5 知所先后，则近道矣

> "物有本末，事有终始，知所先后，则近道矣。"（《大学》）

**采购三阶段关注点变化**：
| 阶段 | 客户关注 | 销售应做 |
|:---:|---------|---------|
| 前期 | 需求、问题、能否解决、如何解决 | 了解概念 |
| 中期 | 方案可行性、可实现愿景 | 呈现优势 |
| 后期 | 价格、风险 | 消除顾虑 |

**比喻**：做项目好比煲汤，什么时候用慢火、快火、加什么佐料都很有讲究。腌黄瓜和黄瓜蘸盐原料一样，工艺不同，结果相差甚远。**只有在合适的时间做合适的事，才能保证最后的胜利。**

#### 3.6 势有必至，理有固然

- **"销售从来都是人卖给人"** — 最终决策拍板的还是个人
- **"所有的购买都是感性决策，决策后再找价值和逻辑使决策看上去很合理"**
- **结果 vs 赢**：结果是冰山上的（提升效率/降低成本），赢是冰山下的（个人利益/职业发展）
- "打土豪，分田地" — 打土豪是革命需要（结果），分田地满足个人需要（赢）
- 满足客户的"结果"是基本条件，满足每一个参与者的"赢"才是制胜根本
- **再意外的结果都能用参与者的赢来解释**

#### 3.7 忧其所虑，务其所需

- 客户的思维是"管状的" — 只考虑自己的业务和问题
- 钻进客户思维的"管子"：了解客户的概念（前期）→ 了解客户的赢（中期）→ 共同开发解决方案
- **"客户认同的解决方案才是真正的解决方案"**
- 面对竞争对手深度占位的项目："要想与众不同，必先与众趋同" — 先让客户认识到你也能达到对手水平，再寻找差异
- 曾国藩："欲成天下之大事，须夺天下人之心"

#### 3.8 就地取材，善假于物

> "君子生非异也，善假于物也。"（《荀子》）

**资源调用三原则**：
1. **时机要合适**：项目初期调售前讲方案 → 不合适；项目后期安排调研 → 作用不大
2. **匹配要精准**：顾问专家要适合解决客户的具体问题，要考虑与客户社交风格的吻合
3. **善用客户内部资源**：让支持者影响反对者。**让客户帮忙是检验与其合作达到某种境界的标志** — 如果他真心帮你，说明你们是合作经营、目标一致

#### 3.9 信而后立，义然后取

> "时然后言，人不厌其言；乐然后笑，人不厌其笑；义然后取，人不厌其取。"（《论语》）

- 客户签单的根本原因是**信任**。一切对付客户的谋略和技巧都是苍白无力的
- **"不要在客户没准备的情况下试图签约"** — 客户没信任你、还有顾虑时，提出签约就会给客户压力
- 面对竞争对手：过分关注对手 = 承认自己不如对手。正确的做法是深入挖掘客户概念
- **只有客户认知的优势才是优势，只有客户认知的差异才是差异**

> **赢单九问是一个首尾相接的闭环系统。起点是"忠恕"，终点是"信任"。**

### 四、项目温度计 — 完整10级感觉描述

> 来源：《赢单九问》第73-78页详细阐述

**核心原理**：问自己"如果客户现在就做出决策，我的感觉如何？" — 这种最直接、未经加工的感觉，就是项目最真实的温度。

| 温度 | 感觉 | 状态描述 | 策略提示 |
|:---:|------|---------|---------|
| **100-90°** | 陶醉 | 完全在掌握中，没有任何危险 | ⚠️ **危！** 自满是失败的开始，100°和0°其实是一个点 |
| **90-80°** | 幸福 | 优势明显，信心满满 | 保持优势，加速推进 |
| **80-70°** | 安全 | 感觉不错，但还有不确定性 | 关注变化，防患未然 |
| **70-60°** | 舒服 | 比较有利，稍加努力即可 | 巩固优势，补强薄弱 |
| **60-50°** | 还行 | 说不上好也说不上坏 | 问自己："哪里还行？"找到不确定因素 |
| **50-40°** | 顾虑 | 有不踏实的感觉 | 分析问题，制定对策 |
| **40-30°** | 不舒服 | 感觉越来越不对 | 深入排查，果断调整 |
| **30-20°** | 担心 | 明显的危险信号 | 立即刹车，重新评估 |
| **20-10°** | 害怕 | 极度不安，预感不好 | 大幅调整策略或止损 |
| **10-0°** | 恐慌 | 没有任何信心，随时可能丢 | 条条是路 — 否极泰来，考虑资源转移 |

**使用要点**：
- 不要用"止痛药"麻醉自己 — "鸵鸟销售"总享受优势不敢看劣势
- 温度<50° → 问："要使现状有什么改变，才能减少担心和焦虑？"
- 温度>50° → 问："为确保成功，我需要做哪些改变？"
- 温度变化极快 — 三种力量（我们/客户/对手）能让温度从90°瞬间降到20°

### 五、单一竞争陷阱

> 来源：《赢单九问》第69-73页

**核心认知**：只有我们一家参与竞争 ≠ 没有竞争。

**单一竞争中的"隐形对手"**：
1. 客户放弃这次采购的"不作为"（"选择死亡"）
2. 客户将预算移作他用
3. 客户内部团队自行完成

> "当路到尽头的时候，条条都是路。"

**案例**：某项目经过长期运作，客户一把手和主要负责人对我方高度认可，通知签约。签合同前，我方有人以为"单一竞争不可能选择别人"，坚持按高价签约。客户感觉被耍，即刻"报复"— 请来竞争对手，按对手报价直接签约。**这是一个双输的项目。**

**对策**：
- 单一竞争更要照顾客户的情绪和心理
- 不要和客户进行"心理博弈"威胁客户
- 除了我们之外的其他任何选择，都是竞争对手

## 场景1：高层破冰 — EB拒绝见面/无法约见

### 场景识别
- **信号**：秘书/助理反复阻挡、微信已读不回、电话不接、邮件不回复
- **典型位置**：项目初期、我方尚未建立任何信任
- **紧迫度**：⭐⭐⭐⭐⭐（没有EB就等于盲打）

### 策略框架
```
阶段1：定位真正阻力
  └─ 阻挡者是秘书？是中层管理者？还是EB本人授意？

阶段2：借力突破
  ├─ 路径A：同行引荐（最有效） → "XX集团张总推荐我来拜访"
  ├─ 路径B：行业活动偶遇 → 提前准备3分钟电梯演讲
  ├─ 路径C：通过UB或Coach建立间接联系
  └─ 路径D：给EB寄送有价值的内容（行业白皮书/对标分析）

阶段3：一击必中
  └─ 第一次见面必须交付"EB觉得值"的东西，否则没有第二次
```

### 话术模板
```
【发给EB的短信/微信（通过第三者转发最佳）】
"张总您好，我是纷享销客的石磊。最近在帮[同类企业]做[具体项目]， 
发现一个行业共性问题：[一句话痛点]。我整理了一份2页的简析，不知
是否方便发您参考？如果方便，10分钟电话跟您同步一下要点。"
```

### 进度检查
- [ ] 是否识别了真正的阻挡者（不是表面阻挡者）？
- [ ] 是否尝试了至少2条借力路径？
- [ ] 第一次接触的PPP剧本是否准备好？
- [ ] 是否准备了EB一定会感兴趣的"见面礼"（数据/洞察/案例）？

## 场景2：技术攻坚 — TB卡在POC/技术评审

### 场景识别
- **信号**：TB迟迟不给技术结论、反复要求补充材料、提出大量技术疑问
- **核心问题**：TB可能不是真"不懂"，而是"不安全"——怕选错了承担责任
- **紧迫度**：⭐⭐⭐⭐（TB不松口，项目走不动）

### 策略框架
```
诊断：TB的顾虑是技术层面还是政治层面？
  ├─ 技术层面 → 补充技术资料 + POC + 技术交流
  └─ 政治层面 → 帮助TB在内部建立"选我们更安全"的论据

攻克三步：
  ① 理解TB的KPI和风险点（他怕什么？）
  ② 提供"第三方背书"（Gartner报告/行业标准/成功案例）
  ③ 帮TB准备内部汇报材料（让他能轻松说服领导）
```

### 话术模板
```
【与TB沟通时】
"李工，我理解您的谨慎。说实话，如果我是您，我也会把每个细节都搞清楚——
因为这关系到整个系统的稳定性和您的专业声誉。

我建议我们分三步走：
第一步，我把您最关心的5个技术点做成一份对比分析（我方 vs 行业标准）
第二步，安排一次POC，重点验证这5个点
第三步，如果POC结果满意，我帮您准备一份内部评审材料，让您向领导汇报时有据可依"
```

### 进度检查
- [ ] TB的真实顾虑是否已定位（技术 or 政治）？
- [ ] 是否提供了第三方背书材料？
- [ ] 是否承诺帮TB准备内部汇报材料（降低他的风险）？

## 场景3：价格博弈 — 客户压价/比价/要求降价

### 场景识别
- **信号**：客户说"太贵了"、拿来竞品报价单对比、要求打折、威胁选竞品
- **关键判断**：客户是真觉得不值？还是在试探你的底线？
- **紧迫度**：⭐⭐⭐⭐（处理不当 → 利润流失 or 丢单）

### 策略框架
```
阶段1：区分真降价需求 vs 试探性压价
  ├─ 试探性（"你们能不能便宜点"随口一说） → 不降价，强化价值
  └─ 真实性（列出来具体贵在哪里） → 差异化解释 or 调整方案范围

阶段2：价值锚定（不直接回应价格）
  ├─ "您觉得贵在哪方面？" → 让客户具体化
  ├─ "相比什么贵？" → 找出对比基准
  └─ "如果我们不解决这个问题，对您意味着什么？" → 回到价值

阶段3：如果必须降价的正确做法
  ├─ 条件交换：降价 ↔ 客户承诺（首付比例/上线时间/推荐信）
  └─ 方案调整：减功能 ≠ 降价（保持价格，调整服务范围）
```

### 无效降价红线（第九问核心）
```
❌ 客户一压就降 → 客户怀疑"之前想多赚我多少？"
❌ 降价对标对手 → 你掉进了对手的价格锚定
❌ 只降价不加条件 → 客户觉得"降了还能再降"
✅ 让价值与价格对等 → 客户觉得"值这个价"
```

### 话术模板
```
【客户说太贵时】
"王总，我完全理解您对成本的关注。我想先确认一下：
您觉得贵，是相对于什么而言的？是相对预算、相对竞品、还是相对预期收益？"

【如果客户拿出竞品报价】
"感谢您分享这个信息。我们先不讨论价格——
您能告诉我，在XX竞品的方案里，有哪些是您认可但我们在方案里没有覆盖到的？
如果有，我们补充；如果没有，那价格差异就不是问题——因为您在比较不同的东西。"
```

### 进度检查
- [ ] 客户是真觉得不值还是试探？
- [ ] 是否先强化价值再谈价格？
- [ ] 如果降价，是否附带了客户承诺？
- [ ] 是否避免了无效降价陷阱？

## 场景4：输单翻盘 — 客户倾向竞品/已有深度合作方

### 场景识别
- **信号**：关键人明确表态"我们已经选了XX"、竞品已做深度POC、客户直接拒绝见面
- **核心问题**：不要试图正面击败已深度占位的对手，要找"裂缝"
- **紧迫度**：⭐⭐⭐（难度极高，ROI需评估；但一旦翻盘收益巨大）

### 策略框架
```
第一步：冷静评估 — 值不值得翻？
  ├─ 客户预算已锁定 → 翻盘难度+++
  ├─ 竞品已深度绑定（定制开发/数据已导入） → 几乎不可能
  ├─ 竞品只是"先用着"，客户不满意 → 有机会
  └─ 判断工具 → 用WPV模型算一下翻盘概率

第二步：找裂缝（定位竞品的薄弱点）
  ├─ 竞品没覆盖到的需求（功能/服务/行业）
  ├─ 客户内部对竞品有不满的人（找UB/TB中的不满者）
  ├─ 竞品近期负面事件（跳票/服务差/人员变动）
  └─ 组织变化（新领导上任/战略调整 → 重新选型窗口）

第三步："曲线入场"
  ├─ 不直接竞争主战场 → 从边缘需求/补充功能切入
  ├─ 先证明"你可以做到竞品能做到的" → 再找差异化
  └─ 等待竞品犯错 → 做好准备随时切入
```

### 话术模板
```
【面对深度占位的竞品】
"了解，XX公司在这个领域确实做得不错。
不过我想问一个可能冒昧的问题：如果满分是10分，
您对XX目前的方案打几分？这扣掉的几分具体是什么？"

【找到裂缝后】
"您提到的这几点，其实恰好是我们最擅长的。
我建议我们可以先从小范围试用开始——
您不需要更换现有系统，我们做补充和增强。
如果效果好，再考虑更多合作；如果不好，您没有任何损失。"
```

### 进度检查
- [ ] 翻盘概率是否>25%（低于此建议放弃，重新评估资源）？
- [ ] 是否找到了至少1条竞品的"裂缝"？
- [ ] 是否采用了"曲线入场"而非正面硬刚？
- [ ] 是否有耐心等待时机（不要急于求成）？

## 场景5：存量增购 — 已签约客户扩展/升级

### 场景识别
- **信号**：老客户提出新需求、业务扩张需要更多授权、上线效果好想推广
- **关键优势**：已有信任基础、已证实的价值、低获客成本
- **紧迫度**：⭐⭐⭐⭐⭐（存量增购ROI最高，应优先响应）

### 策略框架
```
阶段1：确认增购动机
  ├─ 业务扩张驱动（新工厂/新事业部/新区域 → 自然增购）
  ├─ 满意度驱动（用得好，想扩大范围 → 口碑增购）
  └─ 竞争驱动（有竞品在渗透，客户在比价 → 防守增购）

阶段2：增购方案设计
  ├─ 横向扩展：更多部门/更多用户 → 强调"统一平台"价值
  ├─ 纵向深化：更多功能/更高版本 → 强调"ROI递增"
  └─ 打包优惠：多产品组合 → 避免单品降价，做打包价值

阶段3：防守竞品渗透（最关键）
  ├─ 定期与老客户做"价值回顾"（每季度一次）
  ├─ 建立"高管互访"机制（让我们高管定期见客户高管）
  └─ 做客户的"数字化转型顾问"而不只是"软件供应商"
```

### 话术模板
```
【价值回顾会议开场】
"张总，距离上次跟您汇报已经过去3个月了。
我想花20分钟，跟您同步一下这3个月我们的使用数据：
——活跃用户增长了X%
——处理了XX条业务数据
——帮您节省了XX天人工时间

同时，我也想听听您接下来的规划，看我们在哪些方面可以做得更好。"
```

### 进度检查
- [ ] 增购动机是否已明确（扩张/满意/防守）？
- [ ] 是否做了"价值回顾"（用量化数据说话）？
- [ ] 是否有竞品在渗透老客户（定期巡检）？
- [ ] 增购方案是否与客户年度规划对齐？

## 场景6：竞品反超 — 我方落后如何逆转

### 场景识别
- **信号**：客户决策链多数倾向竞品、竞品已经完成POC或方案评审、我方在后期才介入
- **核心问题**：时间窗口紧迫，必须找到"改变游戏规则"的突破口
- **紧迫度**：⭐⭐⭐⭐⭐（时间最宝贵）

### 策略框架
```
阶段1：快速诊断落后原因（2天内完成）
  ├─ 介入太晚？（客户需求已由竞品定义）
  ├─ 关系断层？（关键人不认识我/不信任我）
  ├─ 方案差距？（竞品方案确实更匹配）
  └─ 价格劣势？（竞品价格更低）

阶段2：改变游戏规则（不是追赶，是换赛道）
  ├─ 重新定义需求 → "您确定现在定义的需求是完整的吗？"
  ├─ 引入新决策者 → 找竞品没覆盖到的EB或UB
  ├─ 制造新的评价维度 → "除了功能和价格，实施风险您评估了吗？"
  └─ 拖延决策节奏 → 为客户争取更多比较时间

阶段3：闪电执行（一旦找到突破口，24小时内必须行动）
  ├─ 方案必须在48小时内出
  ├─ 高层必须在72小时内安排对话
  └─ POC必须在1周内启动
```

### 反超战术库
| 战术 | 适用条件 | 核心动作 | 风险 |
|------|---------|---------|------|
| **重新定义需求** | 客户需求模糊或由竞品单方面定义 | 提出"您有没有考虑过XX维度？" | 客户可能觉得你在搅局 |
| **引入新高管** | 竞品只覆盖了采购/IT层面 | 找到业务线EB，从业务价值切入 | 可能得罪现有对接人 |
| **制造安全顾虑** | 竞品方案有明显风险盲区 | 提出实施/合规/数据安全等问题 | 容易被贴上"攻击竞品"标签 |
| **免费试用切入** | 客户对竞品未100%满意 | 提供小范围免费试用，证明价值 | 资源投入大，需评估ROI |
| **分期签约** | 客户预算有限或决策周期长 | 建议"先做一期试试"降低决策门槛 | 一期太小可能无法体现价值 |

### 话术模板
```
【落后时重新定义需求】
"李总，我研究了你们的选型需求。恕我直言，我发现有一个维度目前的评估
似乎没有覆盖到——就是实施后的持续运营成本。很多客户在选型时只看功能
和价格，但是上线3个月后，真正的成本才开始显现。

我建议在最终决策前，增加一个'3年总拥有成本'的评估维度。
我准备了一份对比，只需要30分钟跟您过一下。"
```

### 进度检查
- [ ] 是否在2天内完成了落后原因诊断？
- [ ] 是否找到了至少1个"改变游戏规则"的突破口？
- [ ] 反超战术是否匹配客户当前状态？
- [ ] 闪电执行是否在72小时内启动？

# 🔥 v12.0 新增四：量化模型升级

## 一、竞品威胁度评分模型（CTM — Competitor Threat Model）

```
CTM = Σ(威胁因子得分 × 权重) × 时效系数
```

| 威胁因子 | 权重 | 评分标准（0-10分） |
|---------|:---:|-----------------|
| **客户关系深度** | 0.30 | 竞品与客户决策链的关系紧密程度 |
| **方案匹配度** | 0.25 | 竞品方案与客户需求的匹配程度 |
| **价格竞争力** | 0.15 | 竞品价格优势程度 |
| **品牌/行业地位** | 0.10 | 竞品在行业内的品牌影响力 |
| **服务覆盖** | 0.10 | 竞品的本地化服务能力 |
| **客户惯性** | 0.10 | 客户与竞品的合作历史和历史依赖 |

| 威胁等级 | CTM分值 | 含义 | 策略 |
|:------:|---------|------|------|
| **极高威胁** | ≥75 | 竞品深度占位，正面击败极难 | 侧翼/渗透策略 |
| **高威胁** | 60-74 | 竞品有明显优势 | 差异化+弱点击破 |
| **中等威胁** | 40-59 | 势均力敌 | 强化优势+加速推进 |
| **低威胁** | 25-39 | 我方占优 | 巩固+防止反转 |
| **微威胁** | <25 | 竞品基本无竞争力 | 加速收割 |

## 二、客户紧迫度评分模型（CUS — Customer Urgency Score）

```
CUS = Σ(紧迫度因子得分 × 权重)
```

| 紧迫度因子 | 权重 | 评分标准（0-10分） |
|-----------|:---:|-----------------|
| **业务影响** | 0.30 | 问题不解决对客户业务的影响程度 |
| **时间压力** | 0.25 | 客户内部设定了明确的时间节点 |
| **预算已锁定** | 0.20 | 客户已预留或正在申请预算 |
| **高层关注** | 0.15 | 高层领导对该项目的重视程度 |
| **触发事件** | 0.10 | 是否存在触发采购的外部事件（合规/审计/扩张） |

| 紧迫度等级 | CUS分值 | 含义 | 策略 |
|:--------:|---------|------|------|
| **紧急** | ≥75 | 客户急需解决，时间窗口短 | 加速推进，资源倾斜 |
| **较急** | 55-74 | 客户有较明确的时间表 | 按计划推进，定期检查 |
| **一般** | 35-54 | 客户有需求但不急切 | 持续培育，等待触发 |
| **不急** | <35 | 客户在观望 | 制造紧迫感 or 长期经营 |
