## 一、GMS客户分层模型（正确定义）

### 三层模型

| 层级 | 名称 | 定义 | 销售模式 |
|------|------|------|---------|
| **G** | Global（战略客户） | 大型龙头企业、上市公司、行业标杆 | 顾问式销售+定制化方案 |
| **M** | Middle（中型客户） | 中型企业、成长型客户 | 标准化产品+部分定制 |
| **S** | Small（中小客户） | 中小型企业、潜在大客户 | 标准化产品+自助服务 |

### GMS与销售资源配置

```
G层客户（20%）→ 占用60%销售资源 → 贡献80%营收
M层客户（30%）→ 占用30%销售资源 → 贡献15%营收
S层客户（50%）→ 占用10%销售资源 → 贡献5%营收
```

### GMS分层的战略意义

1. **差异化服务**：不同层级匹配不同销售模式
2. **资源倾斜**：优质资源向G层集中
3. **成长路径**：S→M→G的培育机制
4. **组织设计**：按GMS分层设计销售团队结构

## 二、五级客户关系模型

### 关系层级

| 级别 | 名称 | 特征 | 销售策略 |
|------|------|------|---------|
| **L1** | 认知级 | 知道我们 | 品牌曝光+线索获取 |
| **L2** | 探索级 | 了解产品 | 方案演示+案例证明 |
| **L3** | 合作级 | 开始采购 | 交付保障+满意度管理 |
| **L4** | 伙伴级 | 深度合作 | 联合创新+战略规划 |
| **L5** | 战略级 | 战略协同 | 高层互访+资源互换 |

### 关系升级路径

```
L1 → L2：内容营销 + 行业活动
L2 → L3：方案验证 + 试点项目
L3 → L4：卓越交付 + 高层关系
L4 → L5：战略对齐 + 联合创新
```

### 各层级关键动作

- **L1→L2**：白皮书下载、 webinar参会、官网浏览
- **L2→L3**：POC验证、参考客户拜访、ROI分析
- **L3→L4**：季度经营回顾、客户成功计划、增值服务
- **L4→L5**：战略规划会、联合Go-to-Market、高管互访

## 三、销售4.0时代组织变革

### 销售模式演进

```
销售1.0（产品驱动）→ 销售2.0（关系驱动）→ 
销售3.0（方案驱动）→ 销售4.0（价值驱动+数字化）
```

### 销售4.0的组织特征

1. **扁平化**：减少管理层级，授权一线决策
2. **专业化**：按行业/客户分层组建专业团队
3. **数字化**：CRM+营销自动化+销售赋能工具
4. **生态化**：内部销售+渠道伙伴+战略联盟

## 四、销售模式选择矩阵

### 四种销售模式

| 模式 | 适用场景 | 组织形态 | 核心能力 |
|------|---------|---------|---------|
| **标品销售** | 标准化产品、短周期 | 电话销售+线上 | 线索转化率 |
| **顾问式销售** | 复杂方案、长周期 | 大客户经理制 | 业务理解+方案能力 |
| **方案销售** | 跨产品线整合 | 方案专家+交付团队 | 整合能力+项目管理 |
| **战略销售** | 战略客户、生态合作 | 战略客户部+生态联盟 | 战略规划+资源整合 |

### 模式选择决策树

```
客户采购金额 < 50万？
  ├─ 是 → 标品销售（线上+电话）
  └─ 否 → 销售周期 > 6个月？
        ├─ 是 → 战略销售（战略客户部）
        └─ 否 → 需要定制开发？
              ├─ 是 → 方案销售（方案专家）
              └─ 否 → 顾问式销售（大客户经理）
```

## 五、销售组织能力体系

### 三层能力模型

```
┌─────────────────────────────────┐
│      战略层：方向感              │
│  市场洞察 + 策略制定 + 资源配置  │
├─────────────────────────────────┤
│      战术层：方法感              │
│  销售流程 + 方法论 + 工具赋能    │
├─────────────────────────────────┤
│      执行层：执行力              │
│  过程管理 + 技能训练 + 激励机制  │
└─────────────────────────────────┘
```

### 组织能力诊断问卷

| 维度 | 诊断问题 | 评分（1-5） |
|------|---------|------------|
| 战略清晰 | 销售团队是否清楚年度重点客户？ |   |
| 流程标准 | 是否有标准化的销售流程SOP？ |   |
| 工具赋能 | 销售是否配备了CRM等数字化工具？ |   |
| 激励有效 | 激励机制是否驱动了正确行为？ |   |
| 人才供给 | 新销售上岗周期是否小于3个月？ |   |

> 总分 < 20分 → 组织能力薄弱，需系统建设

## 六、销售团队协作机制

### 售前-销售-交付铁三角

```
         [客户]
            │
      ┌─────┼─────┐
   [售前专家] [销售经理] [交付经理]
      └─────┼─────┘
            │
         [中台支持]
```

### 协作关键机制

1. **联合拜访**：售前+销售共同拜访关键客户
2. **商机评审**：三级评审（经理→总监→VP）
3. **交付握手**：销售向交付的正式交接机制
4. **客户成功**：交付后转入客户成功团队

## 七、组织设计专家的KNOW HOW

### 何时调用

- 企业销售业绩停滞，怀疑组织结构问题
- 业务扩张需要新建销售团队
- 销售渠道冲突，需要重新设计渠道体系
- 销售模式升级（从标品到解决方案）

### 核心交付物

1. **销售组织架构图**（现行 vs 建议）
2. **GMS客户分层方案**（哪类客户由哪支团队覆盖）
3. **销售流程SOP**（按新模式重新设计）
4. **激励机制方案**（与新模式匹配的考核指标）
5. **组织变革路线图**（分阶段实施计划）
