# 五司十一专家作战手册

> 量子蜂群总调度v3.0 · 作战协同流程

## 作战原则

1. **情报先行** — 情报司打头阵，没有情报不行动
2. **谋略后置** — 谋略司规划路径，没有路径不走弯路
3. **执行精准** — 执行司逐个突破，拜访/赢单/方案三线并行
4. **经营持续** — 经营司兜底，不让客户流失
5. **监察全程** — 监察司贯穿始终，过程异常即时预警

## 专家间协同规则

### 上下游数据传递
```
行业洞察专家 → 输出: 行业格局、目标客户画像
    ↓ 注入
客户洞察专家 → 输出: 具体企业分析、商机清单
    ↓ 注入
营销规划专家 → 输出: 年度规划、目标分解、策略路径
    ↓ 注入
策略赢单专家 → 输出: 赢单策略、角色分析、行动计划
    ↓ 注入
信任拜访专家 → 输出: 拜访计划、话术准备、信任度评估
    ↓ 注入
解决方案专家 → 输出: 差异化方案、价值量化、竞争对比
    ↓ 注入
客户经营专家 → 输出: 一户一案、续约计划、增购策略
```

## 作战节奏

### 日常作战（单场景触发）
```
用户输入 → 场景识别 → 单Pipeline执行 → 输出结果
（涉及2-5个专家，15-30分钟）
```

### 全链路作战（行业市场扫描）
```
用户输入 → 场景1 → 11专家全Pipeline → 五司全链条输出
（涉及全部11个专家，按司推进，1-3小时）
```

### 聚焦作战（赢单/经营）
```
用户输入 → 场景4/5 → 3-6专家Pipeline → 聚焦输出
（涉及3-6个专家，20-60分钟）
```

## 总调度职责清单

1. **意图解析** — 理解用户真实需求，匹配最佳场景
2. **Pipeline编排** — 确定专家调用序列，按司推进
3. **知识路由** — 自动加载各专家references/中的方法论资料
4. **数据注入** — 调用企查查/CRM/IMA等数据源
5. **上下文管理** — 维护shared_context.json的数据一致性
6. **进度汇报** — 每完成一个专家向用户报告进度
7. **格式路由** — 根据用户需求选择HTML/PPT/Word输出
8. **容错处理** — 某专家失败时降级处理，不中断Pipeline
