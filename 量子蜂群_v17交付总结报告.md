## 五、设计哲学总纲（道）

> **v17 不是推翻 v16，是给"会干活的军团"装上十一块新肌肉。**

整条暗线：**把"隐式硬编码"变成"显式可治理"**——

| 维度 | v16（隐式） | v17（显式） |
|------|-----------|-----------|
| 路径 | 硬编码本机 | 环境变量 + 降级 |
| 上下文 | 无限堆 | 预算 + 缓存 |
| 工具 | 隐式调用 | Server Card 发现 |
| 记忆 | 一锅炖 | 四类分治 + 程序记忆可写 |
| 进化 | 没刹车 | 评测门控 + 版本回滚 |
| 检索 | 单跳 | 多跳推理 |
| 模型 | 单选 | 多模型共识 |
| 安全 | 无 | 消毒 + 沙箱 + 签名 |
| 可观测 | 日志 | Trace + Eval + Cost |
| 自治 | 全自动/全手动 | 五级非对称自治 |
| GUI | 无 | Computer Use 降级链 |

**六条核心信念（贯穿 v17）**：
1. 上下文是有限资源，边际收益递减（Anthropic）
2. 自进化的核心是自主性，但"自主改"必须配"自主验"（评测门控是刹车片）
3. 销售洞察的价值不在资料多，在关联深（GraphRAG）
4. 选择是"赌一个对"，综合是"让多个模型互相纠错"（MoA）
5. 自治不是"全自动"，是"该自治时自治、该请示时请示"（HITL 非对称）
6. 没有可观测的 Agent 是黑盒，黑盒不敢用（Trace+Eval+Cost）
