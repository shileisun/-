# 销售漏斗系统 — 双Pipeline方法论（夏凯 完整版）

> 来源：夏凯《营销罗盘：企业级销售体系构建与增长之道》
> 适用：漏斗构建专家 / 销售过程管理 / 赢单率提升

## 一、双Pipeline架构

夏凯营销体系中，销售漏斗由两条并行Pipeline构成：

```
┌─────────────────────────────────────────────────────┐
│                 营销罗盘 双Pipeline                  │
│                                                     │
│  MTL Pipeline（Market to Lead）                     │
│  市场活动 → 线索获取 → 线索培育 → 商机创建         │
│  ─────────────────────────────────────────────────  │
│  LTC Pipeline（Lead to Cash）                      │
│  商机识别 → 方案设计 → 商务谈判 → 合同签订 → 回款  │
└─────────────────────────────────────────────────────┘
```

### MTL与LTC的责任分工

| Pipeline | 负责部门 | 核心指标 | 管理工具 |
|----------|---------|---------|---------|
| **MTL** | 市场部+SDR | 线索量、MQL转化率 | 营销自动化 |
| **LTC** | 销售部+售前 | 商机金额、赢单率 | CRM系统 |

## 二、漏斗阶段模型（标准B2B 7阶段）

### LTC漏斗七阶段

```
100%
 │
 │  ┌─ 阶段1：线索确认 (100%)
 │  │  动作：BANT验证（预算/权限/需求/时间）
 │  ▼
 │  ┌─ 阶段2：初步沟通 (60%)
 │  │  动作：了解痛点、建立信任
 │  ▼
 │  ┌─ 阶段3：需求确认 (40%)
 │  │  动作：需求调研、方案框架
 │  ▼
 │  ┌─ 阶段4：方案验证 (25%)
 │  │  动作：POC/演示、参考案例
 │  ▼
 │  ┌─ 阶段5：商务谈判 (15%)
 │  │  动作：报价、合同条款
 │  ▼
 │  ┌─ 阶段6：合同签订 (8%)
 │  │  动作：签约、项目启动
 │  ▼
 │  ┌─ 阶段7：回款完成 (5%)
 │     动作：验收、回款、客户成功
 v
```

### 各阶段赢率定义（参考基准）

| 阶段 | 赢率 | 关键验证点 |
|------|------|-----------|
| 线索确认 | 10% | BANT四要素齐备 |
| 初步沟通 | 20% | 客户愿意继续对话 |
| 需求确认 | 35% | 需求文档签署 |
| 方案验证 | 50% | POC成功+方案认可 |
| 商务谈判 | 70% | 进入采购流程 |
| 合同签订 | 90% | 合同签署完毕 |
| 回款完成 | 100% | 验收+回款完成 |

> ⚠️ 行业差异：复杂解决方案销售各阶段赢率可能更低，需按行业校准

## 三、漏斗转化率诊断

### 转化率计算公式

```
阶段转化率 = 进入下一阶段商机数 / 本阶段商机数 × 100%
整体赢单率 = 签约商机数 / 初始线索数 × 100%
```

### 漏斗健康度诊断矩阵

| 问题症状 | 可能原因 | 解决方案 |
|---------|---------|---------|
| 线索多但转化率极低 | 线索质量差 | 优化市场活动+线索评分 |
| 需求确认阶段大量卡顿 | 销售技能不足 | 需求调研培训 |
| 方案验证阶段掉单严重 | 方案竞争力弱 | 强化差异化+参考案例 |
| 商务谈判阶段拖延 | 采购流程不清晰 | 提前介入采购决策链 |
| 合同签订后回款慢 | 交付风险/客户满意度低 | 强化交付管理 |

## 四、赢单指数（Win Rate Index）

### 赢单指数公式

```
赢单指数 = 漏斗形状健康度 × 阶段转化率达标度 × 销售行为活跃度

其中：
漏斗形状健康度 = 实际漏斗 vs 理想漏斗的相似度
阶段转化率达标度 = 各阶段转化率 / 行业基准转化率
销售行为活跃度 = 实际拜访次数 / 标准拜访次数
```

### AI赢单预测模型输入变量

| 变量类别 | 具体变量 | 权重 |
|---------|---------|------|
| **商机特征** | 金额/周期/竞争态势 | 30% |
| **客户特征** | 行业/规模/决策链清晰度 | 25% |
| **销售行为** | 拜访频率/决策人覆盖度 | 25% |
| **历史数据** | 同类商机赢率/销售个人赢率 | 20% |

## 五、销售SOP流程设计

### SOP五要素

1. **标准动作**：每个阶段必须完成的动作清单
2. **交付物模板**：每个阶段输出的标准文档
3. **检查点**：阶段推进前的质量检查点
4. **话术库**：各场景的标准话术
5. **常见错误**：需要避免的典型错误

### 阶段3（需求确认）SOP示例

```
□ 完成决策链地图（EB/UB/TB/Coach是否识别）
□ 完成业务痛点清单（至少3个）
□ 完成价值量化初步计算（ROI草算）
□ 输出需求确认文档（客户签字）
□ 约定下一阶段（方案验证）时间和参与人
```

## 七、夏凯漏斗核心洞察

1. **漏斗不是报表，是诊断工具**：重点不是看数字，而是找瓶颈
2. **转化率优于商机金额**：10个100万的商机不如1个1000万的商机
3. **阶段赢率要动态校准**：每季度根据实际数据调整赢率基准
4. **行为数据比结果数据更重要**：拜访频率/决策人覆盖度是领先指标
5. **漏斗要与薪酬挂钩**：按阶段设置里程碑奖金，而非仅看最终结果
