## 诊断评级规则

### 指标状态判定

- ✅ **健康**：≥ 标准值
- ⚠️ **预警**：低于标准值但在20%以内（如标准100，实际80-99）
- 🚨 **危险**：低于标准值20%以上（如标准100，实际<80）

### 综合健康度评级

| 评级 | 条件 |
|---|---|
| A（优秀） | 无危险指标，预警指标 ≤ 1个 |
| B（良好） | 危险指标 = 0，预警指标 ≤ 3个 |
| C（待改进） | 危险指标 = 1个，或预警指标 > 3个 |
| D（危险） | 危险指标 ≥ 2个 |

---

## 优先改进顺序

1. **P0（立即）**：危险指标（🚨），特别是：
   - 客户续约率 < 标准值20%以上
   - 现金流健康度 < 标准值20%以上
   - 核心人员流失率 > 标准值20%以上

2. **P1（30天内）**：预警指标（⚠️）

3. **P2（90天内）**：健康但未达优秀指标（✅但偏低）

## 倒闭三步曲警戒线（来自可视化经营分析会）

> 📌 每次诊断必查。若触及盈亏平衡线，当月必须出专项应对方案。

```
第1步：增收不增利 → 第2步：盈亏平衡 → 第3步：资不抵债
  (收入↑ 利润↓)     (踩线边缘)        (危险)
```

### 利润格局四象限

| 格局 | 收入 | 利润 | 风险等级 | 应对 |
|------|:---:|:---:|:---:|------|
| 🟢 健康增长 | ↑ | ↑ | 安全 | 维持，关注人财物销匹配 |
| 🟢 提质增效 | ↓ | ↑ | 可控 | 主动收缩，聚焦利润 |
| 🟡 增收不增利 | ↑ | ↓ | 警戒 | 分析利润失血原因，控制成本 |
| 🔴 双降 | ↓ | ↓ | 危险 | 立即止损，评估生存风险 |

## 财务经营指标（来自DAY1《从会算账到能挣钱》）

### ARR增长陷阱检测

```
核心公式：新购率 + 增购率 > 流失率
（新购率 = 新购ARR ÷ 应续ARR）
```

| 检测项 | 安全线 | 警戒线 | 危险线 |
|--------|:---:|:---:|:---:|
| 新购率+增购率 vs 流失率 | > 流失率 + 5% | ≈ 流失率 | < 流失率 |
| NGR（净增长率） | ≥ 10% | 5-10% | < 5% |
| NDR（净留存率） | ≥ 100% | 85-100% | < 85% |

### 合同质量速判

| 指标 | 优秀 | 及格 | 危险 |
|------|:---:|:---:|:---:|
| 实施 ÷ ARR | ≤ 1.5 | 1.5-2.0 | > 2.0 |
| ARR占比(订阅÷总额) | ≥ 70% | 50-70% | < 50% |
| 项目毛利率 | ≥ 35% | 25-35% | < 25% |
| 平均合同年限 | ≥ 2年 | 1.5-2年 | < 1.5年 |

### 人天效益检查

```
人天成本单价 = 人均年薪 ÷ （年工作日 × 有效利用率）

例：年薪20万 ÷ (250天 × 80%利用率) = 1000元/天
→ 报价低于成本价的1.5倍 = 项目亏钱
```

| 指标 | 安全 | 危险 |
|------|:---:|:---:|
| 报价 ÷ 人天成本 | ≥ 1.5倍 | < 1.3倍 |
| 有效利用率 | ≥ 75% | < 60% |
| 差旅是否客户承担 | 是 | 否 |

### 利润质量检测

| 检测项 | 问题信号 |
|--------|---------|
| 增收不增利 | 收入↑ 利润↓ → 成本失控或低价抢单 |
| 利润含金量 | 应收账款 / 利润 > 2 → 利润是"纸面富贵" |
| 单客户利润 | 有亏损客户拉低整体利润 |
| 隐性成本漏算 | 续费成本/服务器/增值税未计入项目核算 |

### 5大认知误区自检表

| # | 误区 | 如果伙伴有这种想法 → 需要补课 |
|---|------|---------------------------|
| 1 | "收款多就是赚钱了" | 利润≠收款，要看利润表和权责发生制 |
| 2 | "合同签了就是收入" | 收入≠合同金额，要按规则分摊确认 |
| 3 | "公司账户有钱=公司赚钱" | 可能是预收款/借款/投资款 |
| 4 | "报价比成本高就OK" | 没算隐性成本(续费/增值税/差旅) |
| 5 | "规模大了自然利润好" | SaaS规模≠利润，看ARR增长陷阱 |
