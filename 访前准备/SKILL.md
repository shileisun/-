# 访前准备 v1.0

## 定位

在量子蜂群Pipeline中，访前准备位于「策略赢单专家」或「客户洞察专家」之后、「信任拜访专家」之前——承接上游的角色分析结果，系统性完成拜访前的各项准备工作。

```
专家赢单 LTC 全链路：
  客户洞察专家 → 策略赢单专家 → 访前准备 → 信任拜访专家 → 解决方案专家
```

# 第一部分：访前准备清单

> 访前准备的核心是「把不确定性降到最低，把可控性提到最高」。

## 一、客户信息收集清单

### 1.1 基础信息

| 字段 | 收集内容 | 重要性 |
|------|---------|:------:|
| **客户名称** | 全称 + 简称 | ★ |
| **所属行业** | 离散制造/流程制造/装备制造/汽配Tier1/Tier2 | ★ |
| **企业规模** | 人数、产值、分子公司数 | ★ |
| **地理位置** | 总部地址、工厂地址、拜访地点 | ★ |
| **决策链结构** | 老板/事业部/采购/技术/使用分层 | ★ |

### 1.2 客户背景

| 字段 | 收集内容 | 重要性 |
|------|---------|:------:|
| **业务模式** | MTO/MTS/ETO（按单/备库/工程） | ★★ |
| **核心产品** | 主要产品线、市场定位 | ★★ |
| **销售渠道** | 直营/经销/主机厂配套 | ★ |
| **竞争格局** | 主要竞争对手、市场份额 | ★ |
| **近期动态** | 投资/并购/组织调整/战略方向 | ★★ |

### 1.3 关键人信息

| 字段 | 收集内容 | 重要性 |
|------|---------|:------:|
| **拜访对象** | 姓名、职位、背景 | ★★ |
| **决策角色** | E-B/T-CB/T-AB/UB/EK | ★★ |
| **态度倾向** | G/T/EK（支持/中立/不明确） | ★★ |
| **支持度** | 1-5分 | ★★ |
| **个人画像** | 背景经历、决策风格、关注点 | ★★ |

### 1.4 项目信息

| 字段 | 收集内容 | 重要性 |
|------|---------|:------:|
| **需求背景** | 为什么要做？触发点是什么？ | ★★ |
| **预算情况** | 有预算？预算范围？ | ★★ |
| **时间节点** | 决策时间、上线时间 | ★★ |
| **竞争态势** | 有哪些竞争对手？在谈什么阶段？ | ★★ |
| **决策流程** | 几步？谁能拍板？ | ★★ |

## 二、行业知识准备（制造业/汽配）

### 2.1 汽配行业核心知识

| 知识维度 | 准备内容 | 用途 |
|---------|---------|------|
| **Tier体系** | Tier1（一级供应商）/Tier2/Tier3分级 | 理解客户地位 |
| **主机厂配套** | 配套哪些主机厂？主机厂要求？ | 理解业务来源 |
| **年降机制** | 年降压力、成本控制要求 | 理解客户痛点 |
| **JIT模式** | 准时制配送、库存压力 | 理解运营模式 |
| **PPAP流程** | 生产件批准程序、准入门槛 | 理解准入难度 |

### 2.2 离散制造核心知识

| 知识维度 | 准备内容 | 用途 |
|---------|---------|------|
| **BOM结构** | 物料清单、层级复杂度 | 理解产品 |
| **工艺路线** | 生产工序、产能配置 | 理解能力 |
| **工单管理** | 派工、报工、进度 | 理解执行 |
| **批次追溯** | 质量追溯、来源追溯 | 理解质量要求 |
| **项目制销售** | 非标订单、方案设计 | 理解销售模式 |

### 2.3 常见痛点对照

| 行业 | 核心痛点 | 访前需要了解的细节 |
|------|---------|----------------|
| **汽配Tier1** | 主机厂年降、库存高、JIT配送 | 年降比例、库存周转率、配送频次 |
| **汽配Tier2** | 账期长、利润薄、交付难 | 账期、原材料价格、良率 |
| **装备制造** | 项目周期长、应收难、定制多 | 在手项目、回款周期、定制比例 |
| **离散制造** | 多品种小批量、计划难、报工乱 | SKU数、换线时间、报工方式 |

## 三、决策角色分析

### 3.1 五角色关注点

| 角色 | 代码 | 核心关注点 | 访前准备重点 |
|------|:----:|-----------|-------------|
| **经济决策者** | E-B | ROI、战略价值、风险 | 行业案例、ROI量化、高层视角 |
| **技术选型者** | T-CB | 技术架构、产品能力 | 技术方案、稳定性、创新性 |
| **采购执行者** | T-AB | 商务条件、资质 | 价格、商务条款、资质文件 |
| **业务使用者** | UB | 易用性、业务匹配 | 试用体验、用户故事 |
| **行政把关者** | EK | 合规、流程、法务 | 合规文档、流程说明 |

### 3.2 角色态度研判

| 态度 | 信号 | 应对策略 |
|------|------|---------|
| **G（支持）** | 主动联系、积极推荐、内部信息 | 深度信任、价值共创 |
| **T（中立）** | 不拒绝、不推动、还在比较 | 建立认知、消除顾虑 |
| **EK（不明确）** | 态度模糊、可能倾向竞品 | 重新评估、寻找突破口 |

### 3.3 信任缺口分析

```
信任缺口 = 角色关注点 - 我们能提供的价值

分析维度：
1. 角色最在意什么？（关注点）
2. 我们有什么？（价值匹配度）
3. 差距在哪里？（缺口）
4. 怎么补？（突破策略）
```

## 四、拜访目标设定（SSO）

### 4.1 SSO定义

**SSO（Single Sales Objective）单一销售目标**：每次拜访只能有一个核心目标。

### 4.2 SSO完整性检查

| 要素 | 判断标准 | 分值 | 权重 |
|------|---------|:----:|:----:|
| **相关性** | 与客户核心业务目标直接相关 | 0-25 | 25% |
| **清晰度** | 目标具体、可量化、可描述 | 0-20 | 20% |
| **预算** | 有明确预算或预算范围 | 0-20 | 20% |
| **期限** | 有明确的决策时间节点 | 0-15 | 15% |
| **竞争** | 竞争态势清晰、策略明确 | 0-10 | 10% |
| **可及性** | 我们有能力满足需求 | 0-10 | 10% |

### 4.3 常见SSO类型

| SSO类型 | 示例 | 适用场景 |
|--------|-----|---------|
| **获取信息** | 了解客户当前供应商、使用情况 | 初次拜访 |
| **建立认知** | 介绍我们的差异化价值 | 技术交流 |
| **确认需求** | 明确客户的真实需求和预算 | 需求调研 |
| **推动评估** | 安排POC测试或演示 | 方案确认 |
| **获得承诺** | 同意立项/参加演示/提供资料 | 商务推进 |

### 4.4 行动承诺阶梯

```
信息接触 → 进一步了解 → 内部推荐 → 评估参与 → 决策支持
   L1         L2          L3         L4         L5
```

| 层级 | 行动承诺 | 客户行为标志 |
|:----:|---------|-------------|
| L1 | 愿意交流 | 愿意花时间聊 |
| L2 | 给联系方式 | 微信/名片/内部引荐 |
| L3 | 内部推荐 | 把你推给其他人 |
| L4 | 参与评估 | 参加演示/POC测试 |
| L5 | 决策支持 | 在内部推动、提供帮助 |

## 六、话术准备

### 6.1 PPP约见法

| 要素 | 含义 | 示例 |
|:---:|------|------|
| **Purpose** | 一句话说明为什么见面 | "了解您当前的供应商管理情况" |
| **Process** | 说明大致安排 | "先花15分钟了解情况，再用20分钟分享行业实践" |
| **Payoff** | 说明对方能获得什么 | "您可以了解同行是怎么解决类似问题的" |

### 6.2 开场话术模板

```
"您好，我是纷享销客的[名字]。
我们专门服务[制造业/汽配行业]客户，
提供[CRM/供应商管理]解决方案，
主要解决[管理效率/协同响应]问题。

[标杆客户]最近刚好解决了[类似问题]，
[具体效果]。

不知道今天方便了解一下贵司的情况？"
```

### 6.3 异议处理准备

| 异议 | 心理动机 | 应对话术 |
|------|---------|---------|
| "没时间" | 独立性-压力 | "就15分钟，不会耽误您太久" |
| "我们有供应商" | 利己性-风险 | "了解，多一个选择参考" |
| "不需要" | 利己性-认知 | "了解，您现在最关注什么？" |
| "价格太贵" | 利己性-成本 | "理解，先看价值再谈价格" |

### 6.4 逼单 vs 不逼单

```
⚠️ 逼单前提条件：
1. 需求清晰 ✓
2. 方案共识 ✓
3. 没骗没忽悠 ✓

在此基础上，可以轻轻推一把：
"这个月刚好有[优惠/名额]，您看要不要先锁定？"
```

## 七、案例准备

### 7.1 案例筛选原则

| 原则 | 说明 |
|------|------|
| **行业匹配** | 同行业或相近行业 |
| **规模匹配** | 规模相近或更大 |
| **问题匹配** | 解决过类似问题 |
| **效果可量化** | 有具体数据支撑 |

### 7.2 汽配行业标杆案例库

| 案例类型 | 适用场景 | 准备要点 |
|---------|---------|---------|
| **Tier1案例** | 年降压力、主机厂配套 | 库存降低%、响应速度提升 |
| **Tier2案例** | 账期、交付难 | 应收账款周转、交付及时率 |
| **离散制造案例** | 多品种小批量 | SKU管理、计划排程 |
| **装备制造案例** | 项目周期长 | 项目管理、应收管理 |

### 7.3 成功故事公式

```
成功故事 = 行业 + 角色 + 业务痛点 + 解决方案细节 + 价值结果
```

**示例**（汽配行业+采购经理）：
> "说到供应商管理，我们最近刚好服务了一家Tier1企业。
> 他们也是多品种小批量、主机厂JIT要求极高。
> 当时我们一起帮他们搭建了供应商协同管理平台，
> 目前库存周转率提升了30%，响应速度缩短了50%。"
