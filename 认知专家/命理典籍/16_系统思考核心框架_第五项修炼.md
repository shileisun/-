# 系统思考核心框架 · 第五项修炼

> 基于邱昭良《如何系统思考》整理，融入认知专家体系

---

## 一、系统思考 vs 线性思考

| 维度 | 线性思考 | 系统思考 |
|------|---------|---------|
| **基础** | 还原论 | 整体论 |
| **假设** | 问题症状与原因有直接联系 | 原因和结果是多对多、立体、动态的 |
| **方法** | 提升构成部分绩效以改善整体 | 改善各个部分之间的关系以提升整体 |
| **特征** | 静态、片段、机械、关注眼前 | 动态、全局、协调、关注系统整体 |
| **关注点** | 事件 | 结构 |

### 非线性世界的特点

1. **复杂问题的原因多方面**，原因之间并非全部独立或并列
2. **原因和结果是多对多、立体**的，始终处于动态变化中
3. **很多因果之间的联系或反馈是不确定的**，甚至难以预测
4. **不同时刻可能有相对更关键的要素**起主导作用
5. **关键的制约因素是动态变化的**，并非一成不变
6. **事物的发展可能是非线性的**，不一定是匀速的

## 二、思考的罗盘（六步法）

> **从局限于本位到关照全局**

### 核心框架

| 步骤 | 内容 | 问题 |
|------|------|------|
| 1 | 明确关键问题及其衡量指标 | 核心问题是什么？ |
| 2 | 列出完成工作必需的利益相关者 | 谁与这个问题有关？ |
| 3 | 列出各个实体的关键要因或施加的影响 | 他们分别关心什么？ |
| 4 | 列出这些实体受到的影响 | 他们分别受到什么影响？ |
| 5 | 梳理互动关系与反馈回路 | 他们之间是什么关系？ |
| 6 | 分析并确定策略 | 干预点在哪里？ |

### 利益相关者矩阵

```
          │  高影响力  │  低影响力  │
─────────┼───────────┼───────────┤
高重要性 │  关键玩家   │   依赖者   │
         │  (需要管理) │  (需要满足)│
─────────┼───────────┼───────────┤
低重要性 │  满足需求   │   观察者   │
         │  (需要投入) │   (需要监控)│
─────────┴───────────┴───────────┘
```

## 三、因果回路图

> 系统思考的"新语言"

### 核心概念

| 概念 | 定义 | 举例 |
|------|------|------|
| **变量** | 系统中实体的属性、特征或要素 | 销售额、客户满意度 |
| **连接** | 变量之间的关系，有同向(+)和反向(-)两种极性 | 营销投入+→销售量 |
| **回路** | 从一个地方出发，经过一系列环节，最后反作用于自身，形成闭环 | R1、B1 |
| **存量** | 随时间积累的变量 | 客户总数 |
| **流量** | 改变存量的状况 | 新增客户数 |

### 增强回路 vs 调节回路

> **沿着闭合回路完整走一圈，数有多少个"-"型连接**

| 类型 | 定义 | "—"数量 | 效果 |
|------|------|---------|------|
| **增强回路(R)** | 零或偶数个"-"型连接 | 0或偶数 | 每运转一周就增强原有变化趋势 |
| **调节回路(B)** | 奇数个"-"型连接 | 奇数 | 产生与原有变化趋势相反的力量 |

**简化判断**：
- **增强回路**：良性循环 or 恶性循环
- **调节回路**：自我修正 or 恶性循环（过度或不足）

## 四、系统基模

> **复杂背后的简单之美**

### 成长类基模（以推动成长为基础）

| 基模 | 描述 | 结构 | 案例 |
|------|------|------|------|
| **1. 成长引擎** | 自我增强过程，但行动与结果之间有时间延迟 | ���强回路+时间延迟 | 口碑效应：销量→口碑→用户数→销量 |
| **2. 成长上限** | 开始加速增长，然后成长趋缓，逐渐停滞或反复震荡 | 增强回路+调节回路 | 富者愈富：资源有限，两方竞争 |
| **3. 富者愈富** | 两项活动争夺有限资源，成功方获得更多支持 | 增强回路+增强回路 | 直销vs渠道竞争资源 |

### 解决问题类基模（以解决问题为基础）

| 基模 | 描述 | 结构 | 案例 |
|------|------|------|------|
| **1. 延迟反应** | 采取对策，但行动与结果之间有时间延迟，导致过度调整 | 调节回路+时间延迟 | 房价调控：每次都调过头 |
| **2. 饮鸩止渴** | 短期见效的对策，长期产生越来越严重的后遗症 | 调节回路+增强回路 | 价格战：短期提升销量，长期亏损 |
| **3. 舍本逐末** | 治标式方案短期快速见效，但根本性措施用得越来越少 | 调节回路+增强回路 | 应急招聘：临时工多，正式员工少 |

## 五、成长引擎（八种模式）

| 模式 | 回路 | 说明 |
|------|------|------|
| 1. 营销拉动 | 营销投入→销售量→收入→品牌→营销投入 | 持续投入型 |
| 2. 口碑效应 | 销售量→口碑→满意用户→销售量 | 自然增长型 |
| 3. 病毒式营销 | 公众媒体/社交网络→信息传播→新用户 | 传播型 |
| 4. 网络效应 | 用户越多→产品价值越大→新用户 | 平台型 |
| 5. 研发驱动 | 研发→产品竞争力→销量→利润→研发 | 技术型 |
| 6. 人才驱动 | 利润→培训→员工素质→销量→利润 | 团队型 |
| 7. 客户留存 | 留存成本低+满意客户→新客开发 | 存量型 |
| 8. 客单价提升 | 客户购买数量/金额→收入→利润 | 价值型 |

### 设计成长引擎的六步法

1. 列出利益相关者
2. 定义企业对核心利益相关者的价值
3. 发现价值之间的连接
4. 集体研讨，绘制因果回路图草图
5. 预想可能遇到的"成长上限"
6. 检验、优化并确认"成长引擎"

## 六、复杂问题的三种类型

| 类型 | 定义 | 解决思路 |
|------|------|--------|
| **社会性复杂** | 涉及多个利益相关者，关系众多而微妙 | 换位思考、集思广益 |
| **动态性复杂** | 因素众多，彼此错综复杂，因果不在同一时间或空间 | 看懂结构、预判趋势 |
| **涌现性复杂** | 利益相关者模糊，因果难以梳理，未来不确定 | 保持弹性、持续学习 |

### 睿智解决复杂问题的三层境界

1. **精通系统之道**，顺势而为，游刃有余
2. **提前预见**，并采取措施，将问题消除于萌芽状态
3. **问题发生后快速决策**，分清轻重缓急

## 七、系统思考六步法（实践流程）

```
1. 设定问题，描述事件
       ↓
2. 收集相关信息，识别行为模式
       ↓
3. 分析结构，确定变量及连接，绘制因果回路图
       ↓
4. 参考系统基模，找出有效对策
       ↓
5. 反思、浮现并改善心智模式，找出"根本解"
       ↓
6. 采取行动，检验效果
```

## 九、一眼看到本质的检查清单

- [ ] 核心矛盾是什么？
- [ ] 增强回路在哪里？
- [ ] 调节回路在哪里？

- [ ] 系统在成长还是在衰减？

- [ ] 会不会遇到成长上限？

- [ ] 利益相关者是谁？
- [ ] 他们的诉求是什么？
- [ ] 因果回路图画出来
- [ ] 系统基模对应哪个？
- [ ] 干预点在哪里？

## 十、核心金句

> **不是"头痛医头"，是"看到结构"。**

> **不是"最快解决问题"，是"找到根本解"。**

> **不是"预测未来"，是"理解系统"。**

> **格局定层次，矛盾定走向，运势定时机。**

> **知格局者不惑，知矛盾者不惧，知运势者不悔。**
