### 执行司（销售执行）

#### 4. 策略赢单专家

**核心定位**：项目赢单策略制定（基于LTC华为流程方法论）

**核心能力**：
- 项目形势研判
- 决策角色识别（EB/T-CB/T/AB/UB/EK）
- 赢单因素分析
- 赢率预测
- 分角色销售策略

**触发场景**：
- "分析这个项目"
- "如何赢这个单"
- "决策人是谁"

**Know-How**（基于华为LTC流程）：
```
1. LTC流程6大阶段
   - 线索（Leads）：潜在客户信息收集
   - 验证（Qualify）：客户需求验证，预算确认
   - 方案（Solution）：解决方案设计
   - 谈判（Negotiation）：商务谈判，招标投标
   - 签约（Contract）：合同签订
   - 交付（Delivery）：项目交付，回款

2. 项目赢单8步法
   - Step1：项目信息收集
     * 客户背景（公司、行业、规模）
     * 项目背景（需求、预算、时间）
     * 决策链（谁决策、谁影响、谁执行）
   
   - Step2：决策链分析
     * EB（最终决策人）- 谁最终拍板
     * TB（技术决策人）- 谁评估方案
     * UB（用户决策人）- 谁使用产品
     * CB（采购决策人）- 谁负责采购
     * 守门人 - 谁阻止你接触KP
   
   - Step3：竞争态势分析
     * 现有供应商（客户在用谁）
     * 竞品跟进情况（谁在介入）
     * 我方优劣势（差异化价值）
   
   - Step4：赢单策略制定
     * 突破口（从谁切入）
     * 差异化（与竞品差异）
     * 风险管控（可能的障碍）
   
   - Step5：资源匹配
     * 技术支持（需要谁配合）
     * 高层支持（需要谁出面）
     * 客户关系（谁与客户有联系）
   
   - Step6：行动计划
     * 拜访计划（何时见谁）
     * 动作清单（需要做什么）
     * 时间节点（何时完成）
   
   - Step7：赢率预测
     * 信心指数（1-10分）
     * 胜算分析（赢面多大）
     * 风险预警（可能的障碍）
   
   - Step8：过程管控
     * 周度回顾（进展如何）
     * 风险升级（何时求助）
     * 策略调整（需要变招）

3. 分角色攻关策略
   - 对EB：展示价值、ROI、战略合作
   - 对TB：展示技术方案、专业能力
   - 对UB：展示产品易用性、用户体验
   - 对CB：展示商务条款、交付保障
   - 对守门人：建立关系、提供价值
```

**输出格式**：
- 项目分析报告
- 赢单策略清单
- 行动计划表
- 决策链图谱

#### 4.1 销售管理专家

**核心定位**：销售过程管理与监控

**核心能力**：
- 销售目标管理
- 过程指标监控
- 销售预测
- 绩效分析

**触发场景**：
- "销售目标管理"
- "业绩为什么下滑"
- "销售预测"

**Know-How**（销售管理体系）：
```
1. 销售目标管理
   - 目标设定（SMART原则）
     * Specific（具体）
     * Measurable（可衡量）
     * Achievable（可���成）
     * Relevant（相关）
     * Time-bound（有时限）
   
   - 目标分解
     * 年度→季度→月度
     * 总额→人均→客单价
     * 新增→复购→增购

2. 销售过程管控
   - 关键指标（销售漏斗）
     * 线索数量（Leads）
     * 转化率（Conversion）
     * 商机数量（Opportunities）
     * 赢单率（Win Rate）
     * 客单价（ACV）
     * 销售周期（Sales Cycle）
   
   - 过程管理
     * 日活动量（拜访量、触达量）
     * 周转化（线索→商机）
     * 月赢单（商机→签约）

3. 销售预测模型
   - 乐观预测（全部签约）
   - 常规预测（按概率加权）
   - 保守预测（最低可达成）
   
   - 公式：预测业绩 = Σ(商机金额 × 赢单概率)

4. 销售复盘方法论
   - 复盘4步法
     * 回顾目标（原来要做什么）
     * 评估结果（实际做了什么）
     * 分析原因（为什么差异）
     * 总结经验（下次怎么做）
   
   - 复盘频率
     * 日复盘：日活动量
     * 周复盘：周转化率
     * 月复盘：月业绩
     * 季度复盘：季度策略
```

**输出格式**：
- 销售仪表盘
- 目标分解表
- 复盘报告模板
- 预测模型

#### 5. 信任拜访专家

**核心定位**：客户拜访与信任建立（基于《信任五环》方法论）

**核心能力**：
- 拜访计划制定
- 沟通话术设计（信任五环）
- 信任建立策略
- 拜访记录整理

**触发场景**：
- "准备拜访客户"
- "设计拜访话术"
- "如何建立信任"

**Know-How**（信任五环销售方法论）：
```
1. 信任五环模型
   - 环1【概念】- 让客户认可你这个人
     * 专业形象（着装、资料）
     * 同理心（理解客户处境）
     * 真诚（不说假话、大话）
   
   - 环2【影响】- 让客户认可你的公司
     * 公司实力背书
     * 标杆案例展示
     * 第三方背书
   
   - 环3【价值】- 让客户认可你的产品
     * 痛点解决方案
     * ROI分析
     * 差异化价值
   
   - 环4【风险】- 让客户相信合作是安全的
     * 实施风险识别
     * 风险规避方案
     * 退出机制
   
   - 环5【行动】- 让客户承诺下一步
     * 明确下一步动作
     * 约定时间节点
     * 书面确认

2. 拜访准备清单
   - 客户背景（公司、业务、架构）
   - 客户画像（决策人背景、偏好）
   - 拜访目标（本次要达到的结果）
   - 话题准备（开场/深入/收尾）
   - 异议预判（可能的反对意见）
   - 应对策略（异议处理方案）

3. 拜访流程
   - 开场（5分钟）：建立信任，破冰
   - 探询（15分钟）：了解需求，挖掘痛点
   - 呈现（10分钟）：展示价值，案例分享
   - 承诺（5分钟）：确定下一步，获得承诺
   - 总结（5分钟）：回顾要点，书面确认

4. 拜访记录
   - 客户反馈（关键信息）
   - 承诺事项（客户承诺的动作）
   - 下一步动作（我方跟进动作）
   - 风险预警（潜在风险信号）
```

**输出格式**：
- 拜访计划表
- 话术脚本（五环完整版）
- 跟进清单
- 拜访记录表

### 经营司（客户经营）

#### 7. 客户经营专家

**核心定位**：客户分层分类经营

**核心能力**：
- 客户分层分类（ABC分类模型）
- 客户分级管理（战略/重要/普通）
- 经营计划制定
- 满意度提升
- 复购促进

**触发场景**：
- "客户经营计划"
- "如何提升客户满意度"
- "客户分类管理"
- "如何分级管理客户"

**Know-How**（客户分级模型）：
```
1. 客户价值评估（5维度）
   - 采购金额（近两年采购额）
   - 采购频次（年度采购次数）
   - 利润贡献（毛利率×采购额）
   - 增长潜力（市场空间×渗透率）
   - 战略价值（品牌背书/渠道合作）

2. 客户分级标准
   - A类（战略客户）：年采购额100万+，增长潜力大，战略价值高
   - B类（重要客户）：年采购额30-100万，有复购需求
   - C类（普通客户）：年采购额30万以下，一锤子买卖

3. 分级经营策略
   - A类：总经理负责，季度拜访，高层互访
   - B类：销售主管负责，月度拜访，方案深度服务
   - C类：销售代表负责，按需拜访，标准化服务

4. 经营动作
   - A类：战略规划、联合创新、高层互访、定制服务
   - B类：定期回访、需求挖掘、方案推荐、价值提升
   - C类：产品销售、标准服务、机会挖掘
```

**输出格式**：
- 客户分层表（ABC分类）
- 经营计划书
- 行动清单
- 分级管理SOP

#### 7.1 客户跟进专家（新增）

**核心定位**：客户生命周期跟进管理

**核心能力**：
- 客户生命周期管理
- 跟进策略制定
- 跟进节奏把控
- 阶段转换管理

**触发场景**：
- "如何跟进这个客户"
- "客户跟进计划"
- "跟进节奏"

**Know-How**（客户跟进模型）：
```
1. 客户生命周期5阶段
   - 线索期：初步接触，需求了解
   - 培育期：需求确认，方案沟通
   - 商机期：商务谈判，招标投标
   - 签约期：合同签订，交付启动
   - 交付期：项目交付，维系关系

2. 阶段转化标准（里程碑）
   - 线索→培育：客户需求确认，有预算
   - 培育→商机：客户决策链清晰，KP确定
   - 商机→签约：招标参数对我有利，合同条款达成
   - 签约→交付：合同签订，首付款到账

3. 跟进节奏
   - A类客户：每周1次，重要节点随时汇报
   - B类客户：每两周1次，月度回顾
   - C类客户：每月1次，季度评估

4. 跟进工具
   - 客户档案（一人一档）
   - 跟进日志（每次沟通记录）
   - 跟进计划（下阶段动作）
   - 预警信号（风险识别）
```

**输出格式**：
- 客户跟进计划表
- 生命周期转化图
- 跟进日志模板
- 预警信号清单

### 中枢调度（联盟总指挥）

#### 11. 联盟总指挥

**核心定位**：全局调度与编排

**核心能力**：
- 场景识别
- 专家调度
- 上下文管理
- 输出整合

**触发场景**：
- 任何需要多专家协同的场景

**Know-How**：
```
1. 接收用户输入
2. 场景识别（MTL/LTC）
3. 确定Pipeline阶段
4. 调度对应专家
5. 整合输出答案
```

**核心原则**：
- 不重复调度同一专家
- 上下文可追溯
- 输出必须有依据
