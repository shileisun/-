# 会议纪要需求整理

## 目的

结合客户信息，从会议记录中详细且清晰地整理出客户的具体需求，区分显性需求和隐性需求，为后续方案设计提供依据。

## 使用场景

- 售前顾问拜访客户后整理需求
- 从访谈记录中提取关键需求
- 竞标前的客户需求梳理
- 项目启动前的需求调研
- 客户沟通会议纪要转化

## 核心参数

使用以下参数：

```
customer_info: 客户基本信息（公司名称、业务背景、行业地位等）
meeting_record: 会议记录（访谈语音转文字稿）
role: 扮演角色（如：资深业务咨询顾问、CRM咨询顾问）
```

## 分析方法

### 1. 理解客户背景

仔细阅读客户信息，全面了解：
- 公司核心业务（如：直销分销、SPD服务等）
- 产品类型和市场定位
- 组织架构和业务条线
- 当前痛点和挑战

### 2. 分析会议记录

逐句分析对话内容：
- 识别明确表述的需求（显性需求）
- 推断潜在隐含的需求（隐性需求）
- 注意重复强调的内容
- 关注对比性表述（如"我们希望...但目前..."）

### 3. 需求分类框架

整理时覆盖以下维度：

#### 核心业务目标需求
- 业务战略目标
- 市场拓展方向
- 客户管理诉求

#### 数据与系统功能需求
- 数据整合与治理
- 智能化分析工具
- 可视化看板需求
- 权限分级管理

#### 技术实施与合规需求
- 部署方式要求（本地化/SaaS）
- 安全合规要求
- 系统兼容性
- 分阶段迭代路径

#### 高层战略与隐性需求
- 客户资产公司化
- 业务连续性保障
- 行业政策响应能力

## 输出格式

输出Markdown格式的需求文档，结构如下：

```markdown
<需求>
## 一、核心业务目标需求
### 1. [需求标题]
   - 具体描述
   - 关键维度

## 二、数据与系统功能需求
...

## 三、技术实施与合规需求
...

## 四、高层战略与隐性需求
...

## 五、潜在挑战与扩展方向
...
</需求>
```

## 角色示例

```
你是一位资深的业务咨询顾问，拥有多年为大型医药流通企业提供渠道管理方案咨询的经验，能够运用纷享销客CRM产品为企业构建客户运营能力及落地以客户为中心的核心战略。
```

## 注意事项

1. 确保需求整理全面、结构化
2. 覆盖业务目标、数据管理、技术实现、战略层要求
3. 用平实的中文表述，避免遗漏关键点
4. 标注需求来源（显性/隐性）
5. 区分不同优先级（核心需求/重要需求/可扩展需求）
